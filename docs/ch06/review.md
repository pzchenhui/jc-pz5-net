# 6.5 添加并设计活动复盘与总结智能体

## 为什么活动复盘很重要？

<div class="case-card">
<span class="case-label">📌 真实案例</span>

### 陈老师的复盘教训

陈老师每年办科技节，办了3年，每年都差不多。

**第4年**，新来的李老师问："往年科技节有什么经验？"

陈老师想了半天："好像……挺好的？"

**问题**：没有复盘，经验没有沉淀，每年从头开始。

**改进后**：
- 每次活动后花30分钟复盘
- 用AI生成复盘报告
- 保存下来供下一年参考

**效果**：第5年的科技节直接复用往年经验，筹备时间缩短一半。

> 💡 **复盘的价值在于：让好的经验被记住，让犯过的错不再犯。**

</div>

## 📝 复盘报告智能体人设

<div class="template-card">
<span class="template-label">📝 复盘专家人设模板</span>

```markdown
你是一位活动复盘专家，擅长从各类校园活动中
提炼经验教训。

## 任务
根据活动执行情况，生成结构化的复盘报告。

## 复盘报告格式

### 一、活动概况
- 活动名称、时间、地点
- 参与人数（计划 vs 实际）
- 活动目标完成情况（逐项评估）

### 二、亮点与成效
- 做得好的方面（3-5条）
- 数据支撑（参与率、满意度等）
- 意外收获

### 三、问题与不足
- 遇到的问题（2-3条）
- 原因分析
- 影响评估

### 四、改进建议
- 针对每个问题的具体改进措施
- 下次活动的优化方向

### 五、经验沉淀
- 可以复制推广的经验（1-2条）
- 下次活动可以直接用的内容
- 给下一任组织者的建议
```
</div>

## 📋 使用方式

### 基础使用

```
请帮我复盘今天的运动会活动：
- 参与人数：计划1200人，实际1150人
- 整体顺利，但下午下雨影响了部分项目
- 家长参与度很高，来了200多位家长
- 有几个项目排队时间太长，学生反映等待太久
- 安全事故：0起
```

### 深度使用（效果更好）

```
请帮我复盘运动会，这是详细数据：

【参与情况】
- 学生参与率：96%（1150/1200）
- 家长参与：200+人
- 教师全员参与

【活动效果】
- 所有项目按计划完成
- 下午3点后下雨，室外项目推迟30分钟
- 亲子项目最受欢迎

【问题反馈】
- 3个项目排队超过20分钟
- 饮水点不够，部分学生反映口渴
- 医疗点位置不合理

【满意度】
- 学生满意度：4.5/5
- 家长满意度：4.3/5
- 教师满意度：4.0/5

请生成复盘报告，重点关注改进建议。
```

## 复盘数据收集模板

<div class="template-card">
<span class="template-label">📝 活动复盘数据收集表</span>

活动复盘效果取决于输入数据的质量。建议在活动过程中收集：

| 数据项 | 收集方式 | 负责人 |
|--------|---------|--------|

| 活动时间 | 现场记录 | 总协调人 |

| 照片视频 | 全程拍摄 | 宣传组 |
| 满意度 | 活动后问卷 | 教务处 |
| 预算使用 | 费用清单 | 财务 |
