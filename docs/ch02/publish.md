# 2.6 发布：分享与扩散你的智能体

## 发布前检查清单

在点击发布之前，确认以下内容全部通过：

| 检查项 | 状态 |
|--------|------|
| 人设与回复逻辑已完善 | □ |
| 开场白已设置（清晰友好） | □ |
| 推荐问题已设置（3-5个） | □ |
| 知识库已上传（如需要） | □ |
| 插件已配置（如需要） | □ |
| 已进行充分测试（至少5轮） | □ |
| 智能体名称和描述清晰准确 | □ |

## 发布渠道详解

### 1️⃣ Coze 网页——最基础，即时可用

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">发布到网页</span>

1. 在智能体页面点击 **"发布"** 按钮
2. 选择 **"Web App"** 模式
3. 确认后获得分享链接
4. 将链接分享给同事
</div>

**适合场景**：
- 自己日常使用
- 小范围分享给同事
- 快速验证智能体效果

### 2️⃣ 微信公众号——面向家长和学生

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">绑定公众号</span>

1. 在发布页面选择 **"微信公众号"**
2. 按照指引绑定公众号（需要公众号管理员权限）
3. 设置公众号菜单入口（如"学习助手"）
4. 用户在公众号中即可使用智能体
</div>

**适合场景**：
- 面向家长的家校沟通助手
- 面向学生的学科智慧学伴
- 学校活动信息发布助手

### 3️⃣ 飞书——学校办公场景

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">发布到飞书</span>

1. 在发布页面选择 **"飞书"**
2. 授权飞书应用
3. 在飞书中即可对话使用
</div>

**适合场景**：
- 学校使用飞书办公
- 行政助手、会议助手
- 内部工作沟通

### 4️⃣ API 接入——集成到学校系统

适合有技术能力的学校，将智能体嵌入学校现有的OA系统或网站。

## 📋 分享与推广策略

### 在学校内推广

| 方式 | 操作 | 效果 |
|------|------|------|
| 教研活动演示 | 在教研活动中现场演示 | 直观、有说服力 |
| 教师培训 | 纳入教师信息化培训内容 | 系统性推广 |
| 工作群推广 | 在学校工作群分享链接 | 快速覆盖 |
| 一对一帮助 | 帮同事搭建他们的第一个智能体 | 最有效的方式 |
| 使用手册 | 编写简单的使用说明 | 降低使用门槛 |

### 收集反馈

发布不是终点，收集反馈才能持续改进：

| 反馈来源 | 方式 | 关注点 |
|----------|------|--------|
| 同事 | 直接询问使用体验 | 是否好用、有什么痛点 |
| 学生 | 观察使用频率和满意度 | 是否愿意继续用 |
| 使用数据 | 查看平台后台统计 | 使用频次、热门功能 |
| 自我评估 | 定期回顾智能体表现 | 是否需要迭代 |

## 版本管理

智能体可以多次更新发布，建议记录版本号：

| 版本 | 内容 | 日期 |
|------|------|------|
| v1.0 | 初始版本，基础问答 | 2026-05-20 |
| v1.1 | 添加知识库，优化回复格式 | 2026-05-22 |
| v1.2 | 添加插件，支持联网搜索 | 2026-05-25 |
| v2.0 | 添加工作流，支持复杂任务 | 2026-06-01 |

> 💡 **建议**：每次发布前记录版本号，方便追踪变化。

<span class="case-label">📌 推广案例</span>

### 王老师的推广故事

王老师搭建了"数学教案助手"后：

**第1步**：分享给同办公室的2位老师试用
→ 2位老师反馈"挺好用的"

**第2步**：在数学教研组活动中演示
→ 5位老师表示想试试

**第3步**：帮3位老师搭建了她们自己的智能体
→ 数学组有了3个教案助手

**第4步**：整理了一份《智能体使用小贴士》
→ 全组老师都在用

**成果**：从1个人到全组12人，用了 **1个月**。

> 王老师说："分享是最好的学习方式，教别人的同时自己也更清楚了。"

</div>
