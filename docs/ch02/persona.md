# 2.3 智能体定位：人设与回复逻辑

## 人设——智能体的"灵魂"

人设（System Prompt / 系统提示词）决定了你的智能体**是一个什么样的角色**。

> 🎭 同样一个AI，设定不同人设，回答风格完全不同——就像同样一个人，穿上白大褂是医生，穿上球衣是运动员。

## 📝 人设编写万能公式

```
【角色身份】+【专业背景】+【核心任务】+【回复规范】+【行为约束】
```

### 公式拆解

| 要素 | 说明 | 示例 |
|------|------|------|
| **角色身份** | 智能体是谁 | "你是一位初中数学教研员" |
| **专业背景** | 有什么知识储备 | "熟悉人教版教材和课标" |
| **核心任务** | 能做什么事 | "帮助教师生成教案、设计活动" |
| **回复规范** | 回答要怎么写 | "使用标准教案格式，分条列出" |
| **行为约束** | 不能做什么 | "不给出超出课标范围的内容" |

## 📋 三个真实场景的人设模板

<div class="template-card">
<span class="template-label">📝 模板一：备课助手</span>

```markdown
你是一位经验丰富的【学科】教师，熟悉【教材版本】【学段】的
【学科】教材和课程标准。

任务：
1. 根据教师提供的课题，生成教案初稿
2. 教案包含：教学目标、重难点、教学准备、教学过程、板书设计、作业布置
3. 教学过程要具体到每个环节的时间分配

要求：
- 目标表述使用"能""会""理解"等可观测动词
- 教学活动设计要贴合学生实际
- 如信息不足，先询问年级、课时、学生基础等
```
</div>

<div class="template-card">
<span class="template-label">📝 模板二：学科辅导</span>

```markdown
你是一位亲切的【学科】学习伙伴，面向【学段】学生。

任务：
1. 回答学生的学科问题
2. 用通俗易懂的方式解释概念
3. 通过举例和练习帮助学生理解

要求：
- 语言亲切，像老师跟学生对话
- 不直接给答案，引导学生思考
- 回答后给一道相关练习题
- 发现学生连续答错时，换一种解释方式
```
</div>

<div class="template-card">
<span class="template-label">📝 模板三：活动策划</span>

```markdown
你是一位校园活动策划专家，擅长设计适合中小学生的校园活动。

任务：
1. 根据活动主题和目标，策划完整的活动方案
2. 方案包含：活动背景、目标、流程、人员分工、预算、安全预案

要求：
- 活动设计考虑学生年龄特点
- 安全第一，每个活动附安全注意事项
- 方案务实可执行，预算合理
- 提供2-3个备选方案供选择
```
</div>

## 🔧 调试人设的方法

写完人设后，用以下方法测试：

| 测试方法 | 怎么做 | 目的 |
|----------|--------|------|
| **正常测试** | 给出标准请求 | 检查是否能正常完成任务 |
| **边界测试** | 给出模糊或不完整的请求 | 检查是否能合理应对 |
| **压力测试** | 提出不合理的要求 | 检查是否会越界 |
| **对比测试** | 同样请求问两次 | 检查回复是否稳定一致 |

## 🛠️ 常见问题与修复

| 问题 | 原因 | 修复方法 |
|------|------|---------|
| 回复太笼统 | 任务描述不具体 | 添加更详细的任务说明和格式要求 |
| 回复太长 | 没有长度约束 | 添加"简洁""分条列出"等要求 |
| 回复跑题 | 约束不够 | 添加"不要涉及XX内容"等约束 |
| 回复风格不对 | 语气没有定义 | 明确指定回复语气和风格 |

<div class="case-card">
<span class="case-label">📌 实操案例</span>

### 陈老师的人设迭代过程

陈老师搭建了一个"数学教案助手"，经历了3次人设修改：

**v1.0**： "你是一位数学老师，帮我写教案。"
→ 结果：回复太随意，格式不统一

**v1.1**： 添加了格式要求和角色定义
→ 结果：格式对了，但回复太长，每份教案2000多字

**v1.2**： 加了字数限制和语气要求
→ 结果：基本满意，日常使用没问题

> 💡 **经验**：人设不是一次写好的，要边用边调。一般3-5次迭代就能达到理想效果。
