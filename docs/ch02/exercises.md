```
【角色身份】+【专业背景】+【核心任务】+【回复规范】+【行为约束】
```

### 常见错误示例

<div class="case-card">
<span class="case-label">❓ 常见错误示例与修复</span>

#### 错误1：输出格式混乱

**症状**：AI生成的教案没有教学目标，或者各部分内容混在一起。

**原因**：人设中没有明确规定输出格式。

**修复前（人设）**：
```
你是一位数学教师，帮我写教案。
```

**修复后（人设）**：
```
你是一位数学教师。请按以下格式生成教案：
一、教学目标（知识与技能、过程与方法）
二、教学重难点
三、教学过程（导入→新授→练习→总结，每个环节含教师活动、学生活动、设计意图）
四、作业布置
五、板书设计
```

#### 错误2：内容超纲

**症状**：给七年级学生出了九年级难度的题目。

**原因**：人设中没有约束难度范围。

**修复前（人设）**：
```
你是一位数学教师，帮我出题。
```

**修复后（人设）**：
```
你是一位七年级数学教师。所有题目和讲解内容必须符合七年级课标要求，
不得超纲。如涉及不确定内容，请标注"（待确认是否符合课标）"。
```

#### 错误3：语气不当

**症状**：给家长的通知语气太生硬，像在发命令。

**原因**：人设中没有规定语言风格。

**修复前（人设）**：
```
你是一位班主任，帮我写通知。
```

**修复后（人设）**：
```
你是一位小学班主任，擅长家校沟通。撰写通知时：
- 语气亲切、耐心，体现对家长的尊重
- 使用"请您配合""感谢您的支持"等礼貌用语
- 避免命令式语气，多用建议式表达
```

#### 错误4：AI编造数据

**症状**：智能体在通知中编造了不存在的活动时间和地点。

**原因**：人设中没有约束AI不要编造信息。

**修复**：在人设中加入"行为约束"：
```
行为约束：
- 不编造任何数据、时间、地点
- 如果用户没有提供具体信息，请标注"（待补充）"
```

### 评审标准

<div class="template-card">
<span class="template-label">📋 同行评审评分标准</span>

| 评价维度 | 5分（优秀） | 4分（良好） | 3分（合格） | 2分（需改进） | 1分（不合格） |
|---------|-----------|-----------|-----------|-------------|-------------|
| **回复质量** | 输出准确、完整、可直接使用 | 输出基本准确，需小幅修改 | 有基本框架，需修改部分内容 | 存在明显错误，需大幅修改 | 输出完全不可用 |
| **回复速度** | 3秒内响应 | 3-10秒 | 10-20秒 | 20-30秒 | >30秒或经常超时 |
| **格式规范** | 格式完全符合要求 | 格式基本符合，1-2处偏差 | 格式大致正确，多处偏差 | 格式混乱 | 无格式 |
