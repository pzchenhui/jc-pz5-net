# 2.2 抽丝剥茧：从零到一搭建智能体

## 开始搭建！

这一节，你会在 Coze 平台上搭建你的第一个 AI 智能体。别担心，不需要写代码。

## 📋 实操步骤

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">进入创建页面</span>

1. 打开浏览器，访问 [www.coze.cn](https://www.coze.cn)
2. 使用手机号登录（首次需注册）
3. 登录后，点击左上角 **"创建智能体"** 按钮
4. 进入智能体配置页面

你会看到以下几个配置区域：

```
┌─────────────────────────────────┐
│  👤 头像 & 名称                  │
├─────────────────────────────────┤
│  📝 人设与回复逻辑               │
├─────────────────────────────────┤
│  💬 开场白                      │
├─────────────────────────────────┤
│  ❓ 推荐问题                     │
├─────────────────────────────────┤
│  🔌 插件                        │
├─────────────────────────────────┤
│  📚 知识库                      │
├─────────────────────────────────┤
│  🔄 工作流                      │
└─────────────────────────────────┘
```
</div>

<div class="step-card">
<span class="step-number">2</span>
<span class="step-title">给智能体起名</span>

**好名字的标准**：一看就知道是干什么的、简洁好记、带点人情味。

| ❌ 不太好 | ✅ 推荐 | 理由 |
|----------|--------|------|
| AI助手 | 备课小助手 | 一看就知道用途 |
| 教学工具 | 语文教案助手 | 具体到学科 |
| Agent-001 | 智慧学伴·数学 | 有人情味 |

> 💡 **练习**：用一句话描述你想做的智能体，然后提取关键词作为名字。
> 例如："帮我生成初中数学教案" → "初中数学教案助手"
</div>

<div class="step-card">
<span class="step-number">3</span>
<span class="step-title">写一段描述</span>

描述是智能体的"自我介绍"，告诉用户这个智能体能做什么。

<div class="template-card">
<span class="template-label">📝 万能描述模板</span>

```
我是一个[角色]，可以帮你[核心功能1]、[核心功能2]和[核心功能3]。
适用于[适用人群]。
```

**示例：**
```
我是一个初中数学教案助手，可以帮你快速生成教案初稿、
设计课堂活动和编写练习题。适用于初中数学教师。
```
</div>
</div>

<div class="step-card">
<span class="step-number">4</span>
<span class="step-title">选择头像</span>

- 可以在平台内置图库中选择
- 也可以上传自己的图片
- **建议**：用与教育相关的图标，简洁清晰

</div>

<div class="step-card">
<span class="step-number">5</span>
<span class="step-title">保存并发布初版</span>

先完成最基本的配置，发布一个"初版"：
1. 填写**人设与回复逻辑**（下一节详细讲解）
2. 写一句**开场白**
3. 点击 **"发布"**

🎉 **恭喜！你的第一个智能体已经诞生了！**
</div>

## 发布后做什么？

| 步骤 | 操作 | 目的 |
|------|------|------|
| 自己试用 | 问几个问题，看回复怎么样 | 检查基本功能 |
| 请同事试用 | 分享给1-2位同事 | 收集反馈 |
| 持续优化 | 根据反馈调整人设 | 让智能体更好用 |

## 🛠️ 搭建流程图

```
需求分析（2.1节）
    ↓
创建智能体 ← 起名、描述、头像（本节）
    ↓
配置人设（2.3节）
    ↓
添加能力（2.4节）
    ↓
调试优化（2.5节）
    ↓
发布分享（2.6节）
```

<div class="case-card">
<span class="case-label">💡 实操案例</span>

### 刘老师的第一次搭建

刘老师是小学科学教师，她想做一个"科学实验方案助手"。

**她的搭建过程**：
1. 起名："科学实验小帮手"
2. 描述：我是一位小学科学实验设计专家，可以帮你设计适合小学生的科学实验方案。
3. 人设：（暂时没写，先用默认）
4. 发布后试用：输入"设计一个关于植物生长的实验"
5. 结果：AI给出了一个基础方案，但格式不太规范

**她的改进**：
- 第二天回去补充了详细的人设
- 添加了课标知识库
- 再次测试，回复质量大幅提升

> 刘老师的故事告诉我们：**先发布，再优化**，不要追求一次完美。

</div>
