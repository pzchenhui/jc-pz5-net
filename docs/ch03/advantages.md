# 3.2 工作流有何优势？——效率对比与标准化

## 没有工作流 vs 有工作流

<div class="case-card">
<span class="case-label">📌 对比案例</span>

### 场景：准备一堂公开课的完整材料

**没有工作流**——需要手动操作7步：

1. 打开课标知识库 → 搜索对应知识点 → 3分钟
2. 复制课标要求 → 粘贴到对话框 → 1分钟
3. 让AI生成教案 → 等3分钟 → 人工审核 → 5分钟
4. 让AI基于教案出题 → 等2分钟 → 审核答案 → 5分钟
5. 让AI生成课件大纲 → 等1分钟 → 调整格式 → 5分钟
6. 让AI写评课表 → 等1分钟
7. 把所有材料整理到一个文档 → 10分钟

> **总耗时：约 33 分钟，需要7次交互操作**

**有工作流**——只需1步：

1. 输入课题 → 点击运行 → 等待 → 获得完整教学包

> **总耗时：约 5 分钟，只需1次操作**
> ⏱️ **效率提升约 6.6 倍！**

</div>

## 工作流的五大优势（详细说明）

### 1️⃣ 一键执行，减少重复操作

| 场景 | 手动操作次数 | 工作流操作次数 | 节省步骤 | 效率提升 |
|------|-------------|---------------|---------|---------|
| 备课（教案+练习+课件） | 7次 | 1次 | 6步 | 87% |
| 出试卷（选题+答案+排版） | 5次 | 1次 | 4步 | 80% |
| 写周报（数据收集+分析+撰写） | 4次 | 1次 | 3步 | 75% |
| 批量评语（逐个学生生成） | 50次 | 1次 | 49步 | 98% |

**教师场景详解**：

以王老师的数学教案生成为例：

**手动操作**（每周2次）：
1. 打开课标文档 → 查找对应课题（3分钟）
2. 复制课标要求到对话框（1分钟）
3. 输入提示词让AI生成教案（2分钟，等待3分钟）
4. 审核教案，修改不合理的地方（5分钟）
5. 再输入提示词让AI出题（2分钟，等待2分钟）
6. 审核题目和答案（3分钟）
7. 再输入提示词生成课件大纲（1分钟，等待1分钟）
8. 整合所有内容到一个文档（5分钟）

**每次耗时**：约25分钟 × 2次/周 = **每周50分钟**

**工作流操作**：
1. 打开智能体 → 输入课题名称 → 点击运行（1分钟）
2. 等待工作流自动执行（3分钟）
3. 审核输出结果（3分钟）

**每次耗时**：约7分钟 × 2次/周 = **每周14分钟**

**每周节省**：50 - 14 = **36分钟（节省72%）**

### 2️⃣ 流程标准化，保证质量一致性

手动操作时，每次的步骤可能不一样，导致输出质量参差不齐。

工作流确保**每次都按相同的步骤执行**，输出质量稳定。

<div class="template-card">
<span class="template-label">📝 标准化示例（详细步骤）</span>

### 备课工作流的固定步骤

无论输入什么课题，都严格按照以下5步执行，不会遗漏任何环节：

**第1步：查询课标**
- 节点类型：知识库检索
- 输入：课题名称 + 学段
- 操作：从课标知识库中检索对应课题的要求
- 输出：课标要求摘要（知识点、能力目标、教学建议）
- 标准化要点：
  - 检索范围限定为课标文档
  - 返回最相关的3个段落
  - 如果未找到，返回提示"未在课标中找到相关内容"

**第2步：生成教案**
- 节点类型：大模型
- 输入：课题名称 + 学段 + 课标要求（来自第1步）
- 操作：根据课标要求生成完整教案
- 输出：教案（教学目标、重难点、教学过程、作业布置、板书设计）
- 标准化要点：
  - 固定输出格式（5个固定部分）
  - 教学过程包含4个环节（导入、新授、练习、总结）
  - 每个环节标注时间、教师活动、学生活动、设计意图

**第3步：出练习题**
- 节点类型：大模型
- 输入：教案内容（来自第2步）
- 操作：基于教案内容生成练习题
- 输出：基础题5道 + 提高题3道 + 拓展题2道 + 参考答案
- 标准化要点：
  - 固定题量和难度比例
  - 每道题附答案
  - 难度与教案中的教学目标对应

**第4步：做课件大纲**
- 节点类型：大模型
- 输入：教案内容（来自第2步）
- 操作：生成PPT课件大纲
- 输出：10-15页课件大纲（每页含标题、内容、配图建议）
- 标准化要点：
  - 固定页数范围
  - 每页含3个要素（标题、内容、配图）

**第5步：整合输出**
- 节点类型：大模型
- 输入：教案 + 练习题 + 课件大纲
- 操作：整合为完整教学包
- 输出：统一格式的Markdown文档
- 标准化要点：
  - 固定文档结构（教案→练习题→课件大纲）
  - 添加页眉（课题名称 + 学段 + 学科）
  - 使用分隔线区分各部分

> 💡 **关键**：无论谁来操作这个工作流，输出的格式和内容结构都是统一的。这就是标准化的力量。

</div>

### 3️⃣ 降低使用门槛

| 用户类型 | 手动操作需要的能力 | 使用工作流需要的能力 | 门槛降低程度 |
|----------|-------------------|---------------------|------------|
| 新教师（0-3年教龄） | 需要知道课标要求、教案格式、出题技巧 | 只需输入课题 | 大幅降低 |
| 跨学科教师 | 需要学习新学科的教学方法 | 工作流内置流程 | 大幅降低 |
| 行政人员 | 不熟悉教学流程 | 工作流自动完成 | 大幅降低 |
| 老教师 | 可能习惯固定做法但忘记某些步骤 | 工作流不会遗漏 | 适度降低 |

### 4️⃣ 可复用、可分享

工作流一旦搭建好，可以：
- 分享给同事直接使用（一键复制链接）
- 在不同学科间复制修改（如数学教案工作流→语文教案工作流）
- 作为学校的标准流程（统一各学科的备课格式）
- 持续迭代优化（根据使用反馈不断改进）

### 5️⃣ 支持定时和自动触发

工作流不只是手动运行，还可以设置为**自动执行**。

#### 定时执行：让工作流按时运行

<div class="template-card">
<span class="template-label">⏰ 定时执行实操说明</span>

**什么是定时触发？**

定时触发就是让工作流按照你设置的时间自动运行，不需要手动操作。就像闹钟一样，到时间就自动执行。

**Coze平台的定时触发设置步骤**：

1. 进入工作流编辑页面
2. 找到"触发器"或"定时触发"设置（不同版本名称可能不同）
3. 设置触发规则：
   - **一次性**：在指定日期和时间执行一次
   - **周期性**：按天/周/月重复执行
   - **Cron表达式**：高级用户可用Cron语法（如"0 8 * * 1"表示每周一早上8点）
4. 设置执行参数（如果工作流需要输入参数，可以设置默认值）
5. 保存并启用定时触发

**教师常用定时场景**：

| 场景 | 触发时间 | 工作流内容 | 输出 |
|------|---------|-----------|------|
| **每日教学提醒** | 每天早上7:30 | 检查今日课程→生成教学准备清单 | 今日备课提醒 |
| **周报自动生成** | 每周五下午4:00 | 汇总本周教学数据→生成周报 | 本周教学周报 |
| **月度教学总结** | 每月最后一天 | 汇总本月数据→生成月度总结 | 月度教学总结 |
| **学期初计划** | 每学期开学前3天 | 根据课程表→生成学期计划 | 学期教学计划 |

**注意事项**：
- 定时执行需要工作流有默认输入参数（如不需要用户输入）
- 如果工作流需要知识库检索，确保知识库已正确配置
- 定时触发可能受平台免费额度限制，注意监控使用量
- 建议先在手动模式下充分测试，确认无误后再设置定时

</div>

#### 自动触发：事件驱动的自动化

除了定时触发，工作流还可以由特定事件自动触发：

| 触发事件 | 工作流内容 | 应用场景 |
|---------|-----------|---------|
| 收到新邮件 | 自动提取关键信息→生成回复草稿 | 家校邮件自动处理 |
| 表单提交 | 自动汇总表单数据→生成报告 | 学生信息自动汇总 |
| 到达指定日期 | 自动提醒+生成准备材料 | 考试前自动提醒 |

## 工作流 vs 普通智能体：使用场景对比

| 场景 | 适合方式 | 原因 | 示例 |
|------|---------|------|------|
| 回答一个知识问题 | 普通智能体 | 单步任务 | "什么是勾股定理？" |
| 生成一份教案 | 普通智能体 | 单步任务 | "帮我写一份教案" |
| 一键准备完整教学包 | **工作流** | 多步骤，固定流程 | 查课标→教案→练习→课件→整合 |
| 批量生成学生评语 | **工作流** | 重复性多步骤任务 | 为50名学生各生成一条评语 |
| 自动生成教学周报 | **工作流** | 定时执行 | 每周一自动生成 |
| 学生答疑 | 普通智能体 | 交互式对话 | 学生问问题，AI回答 |

## 什么时候该搭建工作流？

当你发现自己在反复做以下事情时，就该考虑搭建工作流了：

| 信号 | 说明 | 举例 |
|------|------|------|
| 每次都要做同样的3-5个步骤 | 适合工作流 | 每次备课都要查课标→写教案→出题 |
| 每周/每月都要做同样的事 | 适合工作流+定时触发 | 每周写教学周报 |
| 不同的老师做同一件事步骤不同 | 适合工作流标准化 | 不同学科老师的教案格式不统一 |
| 新老师不知道怎么做 | 适合工作流降低门槛 | 新教师不知道备课流程 |
| 需要给多人批量处理 | 适合工作流自动化 | 给50名学生写评语 |

## 📋 工作流迁移指南：从手动到自动

如果你已经在用普通智能体做重复性工作，以下是如何迁移到工作流的详细步骤：

<div class="template-card">
<span class="template-label">🔄 迁移指南</span>

### 第1步：识别可迁移的任务

回顾你平时使用智能体的方式，找出以下特征的任务：

- [ ] 你每次都用相同的提示词
- [ ] 你每次都需要手动操作3步以上
- [ ] 你经常需要把A的输出复制给B
- [ ] 你每周/每月都要做同一件事

### 第2步：拆解现有流程

以你平时手动操作的过程为蓝本，拆解为工作流节点：

| 手动步骤 | 对应工作流节点 | 传递变量 |
|---------|--------------|---------|
| 第1步：输入课题 | 开始节点 | topic, grade, subject |
| 第2步：查课标 | 知识库节点 | kb_result |
| 第3步：生成教案 | 大模型节点1 | lesson_plan |
| 第4步：出题 | 大模型节点2 | exercises |
| 第5步：整合 | 大模型节点3 | final_output |

### 第3步：在Coze中搭建

按照[3.3节搭建步骤](/ch03/build-steps)的说明，在Coze中逐个搭建节点。

### 第4步：测试对比

用相同的输入，分别用手动方式和工作流方式各运行一次，对比：
- 操作时间
- 输出质量
- 格式一致性

### 第5步：发布和分享

确认工作流质量达标后，发布并分享给同事使用。

</div>
