# 3.3 工作流的搭建步骤

## 从零搭建一个完整工作流

这一节，你会手把手学会在 Coze 平台上搭建一个"一键备课"工作流。

<div class="case-card">
<span class="case-label">📌 目标</span>

**搭建一个"一键备课"工作流**：
- 输入：学科、年级、课题
- 输出：完整教学包（教案 + 练习题 + 课件大纲）

</div>

## 📋 完整搭建步骤

### 第1步：创建工作流

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">进入工作流页面</span>

1. 登录 Coze 平台
2. 点击左侧导航栏的 **"工作流"**
3. 点击 **"创建工作流"**
4. 填写名称："一键备课工作流"
5. 填写描述："输入课题，自动生成教案、练习题和课件大纲"
6. 进入工作流编辑页面
</div>

### 第2步：设计工作流结构

在动手之前，先用文字画出流程：

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────┐
│  开始    │────→│ 查课标  │────→│ 生成教案 │────→│ 生成练习题│────→│结束 │
│ (输入)  │     │(知识库) │     │ (大模型) │     │ (大模型) │     │     │
└─────────┘     └─────────┘     └─────────┘     └─────────┘     └─────┘
                                                                        ↑
                                              ┌─────────┐              │
                                              │生成课件  │──────────────┘
                                              │(大模型)  │
                                              └─────────┘
```

### 第3步：添加开始节点

<div class="step-card">
<span class="step-number">2</span>
<span class="step-title">配置开始节点</span>

开始节点定义工作流的输入参数：

| 参数名 | 类型 | 说明 | 示例值 |
|--------|------|------|--------|
| `subject` | 文本 | 学科 | "数学" |
| `grade` | 文本 | 年级 | "七年级" |
| `topic` | 文本 | 课题 | "一元一次方程" |

在 Coze 中，点击开始节点，添加以上 3 个参数。
</div>

### 第4步：添加知识库检索节点

<div class="step-card">
<span class="step-number">3</span>
<span class="step-title">连接课标知识库</span>

1. 点击 **"+ 添加节点"**
2. 选择 **"知识库"** 类型
3. 选择你之前上传的课标知识库
4. 设置查询内容：`{{subject}} {{grade}} {{topic}} 课程标准`
5. 设置输出变量名：`standard_info`
</div>

### 第5步：添加教案生成节点

<div class="step-card">
<span class="step-number">4</span>
<span class="step-title">配置大模型节点（教案）</span>

1. 点击 **"+ 添加节点"**
2. 选择 **"大语言模型"** 类型
3. 配置系统提示词：

```markdown
你是一位教案编写专家。请根据以下信息生成一份完整教案：

学科：{{subject}}
年级：{{grade}}
课题：{{topic}}
课标要求：{{standard_info}}

教案格式：
1. 教学目标（三维目标）
2. 教学重难点
3. 教学准备
4. 教学过程（含时间分配）
5. 板书设计
6. 作业布置

要求：目标使用可观测动词，教学过程具体可操作。
```

4. 设置输出变量名：`lesson_plan`
</div>

### 第6步：添加练习题生成节点

<div class="step-card">
<span class="step-number">5</span>
<span class="step-title">配置大模型节点（练习题）</span>

1. 添加 **"大语言模型"** 节点
2. 系统提示词：

```markdown
你是一位命题专家。请根据以下教案内容，生成一套练习题：

教案内容：{{lesson_plan}}

要求：
- 选择题10道（基础6道、中等3道、挑战1道）
- 填空题5道
- 应用题3道
- 附答案和解析
- 总分为100分
```

3. 设置输出变量名：`exercises`
</div>

### 第7步：添加课件大纲生成节点

<div class="step-card">
<span class="step-number">6</span>
<span class="step-title">配置大模型节点（课件）</span>

1. 添加 **"大语言模型"** 节点
2. 系统提示词：

```markdown
你是一位课件设计专家。请根据以下教案，生成PPT课件大纲：

教案内容：{{lesson_plan}}

要求：
- 每页一个知识点
- 标注配图建议
- 标注动画建议
- 总页数15-20页
```

3. 设置输出变量名：`ppt_outline`
</div>

### 第8步：添加结束节点

<div class="step-card">
<span class="step-number">7</span>
<span class="step-title">整合输出</span>

1. 添加 **"结束"** 节点
2. 将三个大模型节点的输出连接到结束节点
3. 定义最终输出格式：

```markdown
## 📋 教案
{{lesson_plan}}

## ✍️ 练习题
{{exercises}}

## 📊 课件大纲
{{ppt_outline}}
```
</div>

## 📋 测试工作流

### 测试步骤

<div class="step-card">
<span class="step-number">8</span>
<span class="step-title">运行测试</span>

1. 点击 **"测试运行"** 按钮
2. 输入测试参数：
   - subject: 数学
   - grade: 七年级
   - topic: 一元一次方程
3. 点击运行
4. 查看每一步的执行结果
</div>

### 调试技巧

| 问题 | 排查方法 |
|------|---------|
| 某节点没执行 | 检查连线是否正确连接 |
| 输出不对 | 检查该节点的输入参数是否正确 |
| 运行报错 | 查看错误日志，定位问题节点 |
| 知识库没返回 | 检查知识库是否上传成功、检索词是否匹配 |

## 工作流设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **清晰** | 每个节点职责单一 | 一个节点只做一件事 |
| **精简** | 不要过度复杂 | 够用就好 |
| **容错** | 考虑异常情况 | 添加备用方案 |
| **可维护** | 给节点命名 | 方便后续修改 |

## 📋 工作流模板库

搭建好一个工作流后，可以复制到其他场景：

| 工作流 | 输入 | 输出 | 复用场景 |
|--------|------|------|---------|
| 一键备课 | 学科+年级+课题 | 教案+练习题+课件 | 各学科通用 |
| 试卷生成 | 范围+难度+题型 | 完整试卷+答案 | 各学科、各年级 |
| 活动策划 | 类型+对象+预算 | 活动方案+文案 | 各类校园活动 |
| 公文写作 | 类型+主题+要素 | 完整公文 | 通知、报告、总结 |
