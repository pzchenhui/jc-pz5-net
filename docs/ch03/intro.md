# 第3章 AI智能体进阶——工作流

## 从「能回答问题」到「自动完成任务」

前两章你已经学会了搭建基础智能体：设定人设、配置能力、发布使用。

但你会发现一个问题：**智能体虽然会回答问题，但只能一次回答一个问题。**

<div class="case-card">
<span class="case-label">📌 真实场景</span>

### 备课 vs 智能体备课

**传统智能体的局限：**

你问智能体："帮我准备三年级数学课"
→ 它给你一份教案
→ 你还需要再问："帮我出5道练习题"
→ 你还需要再问："帮我做课件大纲"

每次都要单独提问，**重复操作3次**。

**有了工作流之后：**

你告诉智能体："帮我准备三年级数学课"
→ 智能体**自动**调用工作流：
  1. 📋 查课标→确定知识点要求
  2. 📝 生成教案→含教学目标、重难点、教学过程
  3. ✍️ 出练习题→按难度梯度
  4. 📊 做课件大纲→每页内容和动画建议
  5. 💾 打包整理→统一格式

**一次操作，全套产出。**

</div>

这就是工作流的力量——**把多步骤任务编排成自动化流程**。

## 什么是工作流？

> 🎯 **工作流（Workflow）** = 一组按照特定顺序执行的节点（步骤），每个节点完成一个子任务，最终实现一个完整的业务流程。

<div class="template-card">
<span class="template-label">🧩 通俗比喻</span>

**工作流就像工厂流水线：**

- 普通智能体 = 一个多能工，一次做一件事
- 工作流 = 一条生产线，原材料进去，成品出来
- 智能体 + 工作流 = 一个有生产线的工厂

你只需要把原材料（需求）放进去，整条生产线自动运转，成品（结果）自动出来。

</div>

## 工作流的核心组件

| 组件 | 说明 | 类比 |
|------|------|------|
| **开始节点** | 工作流的入口，接收用户输入 | 生产线的进料口 |
| **大模型节点** | 调用AI模型生成内容 | 加工车间 |
| **条件分支** | 根据不同条件走不同路线 | 分拣台 |
| **代码节点** | 执行自定义代码处理数据 | 精密仪器 |
| **知识库节点** | 从知识库检索相关信息 | 资料库 |
| **结束节点** | 输出最终结果 | 出料口 |
