# 第3章 AI智能体进阶——工作流：从「能回答问题」到「自动完成任务」

## 本章核心概念

前两章你已经学会了搭建基础智能体：设定人设、配置能力、发布使用。

但你会发现一个问题：**智能体虽然会回答问题，但只能一次回答一个问题。**

<div class="case-card">
<span class="case-label">📌 真实场景</span>

### 备课 vs 智能体备课

**传统智能体的局限：**

你问智能体："帮我准备三年级数学课"
→ 它给你一份教案
→ 你还需要再问："帮我出5道练习题"
→ 你还需要再问："帮我做课件大纲"

每次都要单独提问，**重复操作3次**。

**有了工作流之后：**

你告诉智能体："帮我准备三年级数学课"
→ 智能体**自动**调用工作流：
  1. 📋 查课标→确定知识点要求
  2. 📝 生成教案→含教学目标、重难点、教学过程
  3. ✍️ 出练习题→按难度梯度
  4. 📊 做课件大纲→每页内容和动画建议
  5. 💾 打包整理→统一格式

**一次操作，全套产出。**

</div>

这就是工作流的力量——**把多步骤任务编排成自动化流程**。

---

## 什么是工作流？

> 🎯 **工作流（Workflow）** = 一组按照特定顺序执行的节点（步骤），每个节点完成一个子任务，最终实现一个完整的业务流程。

<div class="template-card">
<span class="template-label">🧩 通俗比喻</span>

**工作流就像工厂流水线：**

- 普通智能体 = 一个多能工，一次做一件事
- 工作流 = 一条生产线，原材料进去，成品出来
- 智能体 + 工作流 = 一个有生产线的工厂

你只需要把原材料（需求）放进去，整条生产线自动运转，成品（结果）自动出来。

</div>

## 工作流的核心组件（详细说明）

| 组件 | 说明 | 类比 | 教师场景示例 |
|------|------|------|------------|
| **开始节点** | 工作流的入口，接收用户输入的参数 | 生产线的进料口 | 输入课题名称"分数的认识"、学段"小学" |
| **大模型节点** | 调用AI大模型生成内容，是工作流的"大脑" | 加工车间 | 根据课标要求生成教案，根据教案生成练习题 |
| **条件分支** | 根据不同条件走不同路线，实现判断逻辑 | 分拣台 | 如果学段是"小学"→生成游戏化教案；如果是"高中"→生成深度探究教案 |
| **代码节点** | 执行自定义代码处理数据，适合格式转换、计算等 | 精密仪器 | 将多个节点的输出整合为Markdown文档，计算日期差 |
| **知识库节点** | 从预先上传的知识库中检索相关信息 | 资料库 | 从《义务教育课程标准》中检索对应课题的要求 |
| **结束节点** | 输出最终结果给用户 | 出料口 | 输出完整的教学包（教案+练习题+课件大纲） |

### 核心组件详细说明

<div class="case-card">
<span class="case-label">📌 组件详解</span>

#### 1. 开始节点

开始节点是整个工作流的入口。你需要在这里定义用户输入什么参数。

**关键配置项**：
- **参数名**：变量的名称，如 `topic`、`grade`、`subject`
- **参数类型**：字符串（文本）、数字、选项（下拉选择）
- **是否必填**：哪些参数是必须的，哪些是可选的
- **默认值**：如果用户不输入，使用什么默认值

**教师场景示例**：
- 参数1：`topic`（课题名称），字符串，必填 → 用户输入"分数的认识"
- 参数2：`grade`（学段），选项[小学/初中/高中]，必填 → 用户选择"小学"
- 参数3：`subject`（学科），字符串，必填 → 用户输入"数学"

**注意事项**：
- 参数名用英文，避免中文变量名导致的兼容问题
- 必填参数不要太多，否则用户输入太复杂
- 选项类型适合固定范围的选择（如学段、难度）

#### 2. 大模型节点

大模型节点是工作流的"大脑"，负责调用AI模型生成内容。

**关键配置项**：
- **模型选择**：GPT-4o、Claude 3.5 Sonnet、文心一言等
- **系统提示词**：定义AI的角色和约束（即"人设"）
- **用户提示词**：具体的任务指令，可以使用 `{{变量名}}` 引用上游变量
- **温度（Temperature）**：控制输出的创造性（0=确定性最强，1=最自由）

**教师场景示例**：
```
系统提示词：你是一位有10年经验的小学数学教师，擅长教案编写。

用户提示词：
请根据以下课标要求为{{grade}}{{subject}}编写一份关于"{{topic}}"的教案：
课标要求：{{kb_result}}

输出格式：
一、教学目标
二、教学重难点
三、教学过程
四、作业布置
```

**注意事项**：
- 提示词要具体，告诉AI输出什么格式
- 使用 `{{变量名}}` 语法引用上游节点的输出
- 温度设置为0.3-0.7（太低太死板，太高太随机）

#### 3. 条件分支

条件分支让工作流有了"判断"能力，可以根据输入走不同的路线。

**关键配置项**：
- **判断条件**：如果X等于/包含/大于Y，走路线A；否则走路线B
- **分支数量**：通常2-3个分支，太多会导致维护困难

**教师场景示例**：
```
条件：input.grade 的值
- 如果等于"小学" → 走路线A（生成游戏化教案）
- 如果等于"初中" → 走路线B（生成标准教案）
- 如果等于"高中" → 走路线C（生成深度探究教案）
```

#### 4. 代码节点

代码节点适合需要精确处理数据的场景，如格式转换、数据计算。

**教师场景示例**：
- 将多个节点的输出整合为一份Markdown文档
- 计算两个日期之间的天数
- 格式化数字（如将0.85转换为85%）

**注意**：不需要编程基础的教师可以跳过这个节点，用大模型节点替代。

#### 5. 知识库节点

知识库节点让工作流能"查资料"，是智能体专业性的关键。

**教师场景示例**：
- 上传《义务教育课程标准2022版》，让AI查课标要求
- 上传历年优秀教案，让AI参考格式和风格
- 上传学校规章制度，让AI遵守学校规定

#### 6. 结束节点

结束节点决定工作流最终输出什么给用户。

**关键配置**：选择从哪个节点取输出作为最终结果，或者将多个节点的输出整合后输出。

</div>

### 操作方式

- **添加节点**：从左侧工具栏拖拽到画布，或点击画布中的"+"按钮
- **连接节点**：从一个节点的输出口拖拽到另一个节点的输入口
- **编辑节点**：点击节点，在右侧属性栏中配置
- **测试运行**：点击右上角"测试"，输入参数查看运行结果
- **发布**：点击右上角"发布"，使工作流可用

## 工作流 vs 普通智能体：使用场景对比

| 场景 | 适合方式 | 原因 | 示例 |
|------|---------|------|------|
| 回答一个知识问题 | 普通智能体 | 单步任务，不需要流程 | "什么是勾股定理？" |
| 生成一份教案 | 普通智能体 | 单步任务 | "帮我写一份教案" |
| 一键准备完整教学包 | **工作流** | 多步骤，固定流程 | 查课标→教案→练习→课件→整合 |
| 批量生成学生评语 | **工作流** | 重复性多步骤任务 | 为50名学生各生成一条评语 |
| 自动生成教学周报 | **工作流** | 定时执行 | 每周一自动生成 |
| 学生答疑 | 普通智能体 | 交互式对话 | 学生问问题，AI回答 |
| 公文审核 | **工作流** | 条件判断+多步分析 | 格式审核→内容审核→输出报告 |

## 📌 教师场景示例：三个典型工作流

<div class="case-card">
<span class="case-label">📌 教师场景示例</span>

### 场景1：数学教师的"一键出题"工作流

**痛点**：每周要出一次周测，从选题到排版约需2小时。

**工作流设计**：
```
开始（输入：章节名称、难度、题量）
  ↓
知识点检索 → 从教材知识库中提取相关知识点
  ↓
题目生成 → 按难度和题型要求生成试卷
  ↓
答案生成 → 为每道题生成标准答案
  ↓
格式整合 → 整合为可打印的A4格式
  ↓
结束（输出：完整试卷+答案）
```

**效果**：出题时间从2小时→15分钟。

### 场景2：班主任的"批量评语"工作流

**痛点**：期末要给50名学生写个性化评语，每条约5分钟，总计4小时。

**工作流设计**：
```
开始（输入：学生名单CSV：姓名+关键词）
  ↓
数据解析 → 读取CSV，提取学生信息
  ↓
循环处理 → 为每个学生生成评语
  ↓
格式整合 → 汇总为评语表
  ↓
结束（输出：完整评语表）
```

**效果**：评语撰写时间从4小时→30分钟。

### 场景3：教务主任的"数据报表"工作流

**痛点**：每月汇总各部门数据制作运行报告，约需2天。

**工作流设计**：
```
开始（输入：各部门数据表格）
  ↓
数据提取 → 从各表格中提取关键指标
  ↓
数据分析 → 生成同比/环比分析文字
  ↓
报告生成 → 整合为月度运行报告
  ↓
结束（输出：完整运行报告）
```

**效果**：数据汇总时间从2天→3小时。
