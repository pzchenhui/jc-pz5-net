# 3.4 智能体如何调用工作流？

## 工作流建好了，怎么让智能体用上？

有两种方式让智能体调用工作流：

## 方式一：作为插件调用（推荐）

这是最简单的方式——将工作流发布为插件，智能体自动识别并调用。

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">将工作流发布为插件</span>

1. 在工作流页面，点击 **"发布"**
2. 选择 **"发布为插件"**
3. 填写插件名称和描述
4. 确认后，工作流就变成了一个可用插件
</div>

<div class="step-card">
<span class="step-number">2</span>
<span class="step-title">在智能体中添加工作流插件</span>

1. 回到智能体配置页面
2. 在"插件"模块中，搜索你刚发布的工作流插件
3. 点击 **"添加"**
4. 保存智能体
</div>

### 效果演示

<div class="case-card">
<span class="case-label">📌 调用过程</span>

**用户输入**：
> 帮我准备七年级一元一次方程的完整教学材料

**智能体内部流程**：
1. 识别到用户需要"完整教学材料"
2. 匹配到"一键备课工作流"插件
3. 提取参数：学科=数学、年级=七年级、课题=一元一次方程
4. 调用工作流
5. 将工作流返回的结果整理后展示给用户

**用户看到的结果**：
```
好的！我已为你生成七年级数学"一元一次方程"的完整教学材料：

## 📋 教案
（工作流返回的教案内容）

## ✍️ 练习题
（工作流返回的练习题）

## 📊 课件大纲
（工作流返回的课件大纲）
```

</div>

## 方式二：在智能体人设中指定

这种方式更灵活，可以精确控制什么时候调用工作流。

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">在人设中添加调用指令</span>

```markdown
当用户要求生成完整教学包时，请按以下流程执行：

1. 调用"备课工作流"，传入用户的学科、年级和课题
2. 将工作流返回的结果整理后展示给用户
3. 如果用户只需要教案，不要调用工作流，直接生成即可
```
</div>

### 两种方式对比

| 方式 | 优点 | 缺点 | 适合场景 |
|------|------|------|---------|
| **插件调用** | 智能体自动判断何时调用 | 可能误判（不该调用的时候调用了） | 功能单一、触发条件明确 |
| **人设指定** | 精确控制调用条件 | 需要手动编写规则 | 条件复杂、需要判断 |

## 📋 常见问题与解决

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 工作流没有被调用 | 智能体没识别到触发条件 | 在人设中明确调用条件 |
| 调用报错 | 工作流输入参数不匹配 | 检查参数名称和类型是否一致 |
| 结果不完整 | 工作流某节点执行失败 | 单独测试工作流，定位问题节点 |
| 调用了不该调的 | 智能体误判 | 在人设中添加更明确的触发条件 |

## 工作流调用的最佳实践

<div class="template-card">
<span class="template-label">📝 人设中的调用规则示例</span>

```markdown
## 工作流调用规则

当用户的请求符合以下条件时，调用对应的工作流：

1. 用户说"完整教学包""全套材料""一键备课"等关键词
   → 调用"一键备课工作流"

2. 用户说"出试卷""组卷""测试题"等关键词
   → 调用"试卷生成工作流"

3. 用户说"策划活动方案""活动方案"等关键词
   → 调用"活动策划工作流"

其他情况，正常对话回答即可。
```

</div>
