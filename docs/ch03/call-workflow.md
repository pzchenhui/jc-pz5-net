# 3.4 智能体如何调用工作流？——让工作流真正"活"起来

工作流建好了，怎么让智能体用上？有两种方式让智能体调用工作流：

```
方式一：作为插件调用（推荐）→ 简单、自动
方式二：在智能体人设中指定 → 灵活、精确
```

> ⏱️ **预计阅读用时**：约15分钟 | **本节产出**：掌握两种调用方式，让智能体自动触发工作流

## 方式一：作为插件调用（推荐）

这是最简单的方式——将工作流发布为插件，智能体自动识别并调用。

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">将工作流发布为插件</span>

1. 在工作流页面，点击 **"发布"**
2. 选择 **"发布为插件"**
3. 填写插件名称和描述（如"一键备课：输入课题，自动生成教案+练习题+课件大纲"）
4. 确认后，工作流就变成了一个可用插件

<span class="step-number">2</span>
<span class="step-title">在智能体中添加工作流插件</span>

1. 回到智能体配置页面
2. 在"插件"模块中，搜索你刚发布的工作流插件
3. 点击 **"添加"**
4. 保存智能体
</div>

### 效果演示

<div class="case-card">
<span class="case-label">📌 调用过程</span>

**用户输入**：
> 帮我准备七年级一元一次方程的完整教学材料

**智能体内部流程**：
1. 识别到用户需要"完整教学材料"
2. 匹配到"一键备课工作流"插件
3. 提取参数：学科=数学、年级=七年级、课题=一元一次方程
4. 调用工作流
5. 将工作流返回的结果整理后展示给用户

**用户看到的结果**：
```
好的！我已为你生成七年级数学"一元一次方程"的完整教学材料：

## 📋 教案
（工作流返回的教案内容）

## ✍️ 练习题
（工作流返回的练习题）

## 📊 课件大纲
（工作流返回的课件大纲）
```

> 💡 **关键**：用户只需要说一句话，智能体就自动完成了原本需要7步操作的任务。

</div>

## 方式二：在智能体人设中指定

这种方式更灵活，可以精确控制什么时候调用工作流。适合条件复杂、需要判断的场景。

### 完整操作步骤

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">在工作流页面发布工作流</span>

1. 在工作流页面，点击 **"发布"**（不需要选"发布为插件"）
2. 确认工作流状态为"已发布"

<span class="step-number">2</span>
<span class="step-title">在智能体中关联工作流</span>

1. 回到智能体配置页面
2. 在"能力"或"工作流"模块中，添加已发布的工作流
3. 保存智能体

<span class="step-number">3</span>
<span class="step-title">在人设中编写调用规则</span>

在智能体的"人设与回复逻辑"配置框中，添加工作流调用规则。完整示例见下方。

<span class="step-number">4</span>
<span class="step-title">测试调用</span>

- 输入"帮我备七年级一元一次方程的课" → 应调用工作流
- 输入"帮我出一道数学题" → 不应调用工作流，直接回答
</div>

### 完整人设配置示例

以下是一个"全能备课助手"的完整人设，包含多工作流调用规则。你可以直接复制使用：

<div class="template-card">
<span class="template-label">📝 完整人设配置示例（含多工作流调用规则）</span>

```markdown
你是一位有15年经验的全科备课专家，擅长教案设计、试题命制
和课件制作。

## 你的能力
1. 教案生成
2. 试题生成
3. 课件大纲提取
4. 教学包整合

## 工作流调用规则

当用户的请求符合以下条件时，调用对应的工作流：

### 场景1：完整教学包
当用户说"完整教学包""全套材料""一键备课""帮我备XX课"等关键词时：
→ 调用"一键备课工作流"
→ 输入参数：学科、年级、课题
→ 输出：教案+练习题+课件大纲

### 场景2：试卷生成
当用户说"出试卷""组卷""测试题""月考"等关键词时：
→ 调用"试卷生成工作流"
→ 输入参数：学科、年级、考试范围、题型分布
→ 输出：完整试卷+答案+评分标准

### 场景3：活动策划
当用户说"策划活动方案""活动方案"等关键词时：
→ 调用"活动策划工作流"
→ 输入参数：活动类型、目标人群、预算
→ 输出：活动方案+海报文案+执行清单

### 场景4：单独任务
当用户只需要单一任务时（如"帮我出一道题""帮我写教案"）：
→ 不调用工作流，直接生成

### 场景5：不确定时
如果不确定是否应该调用工作流：
→ 先询问用户"您需要完整教学包还是单一任务？"
→ 根据用户回答决定是否调用

## 回复规范
- 调用工作流后，将结果整理为清晰的格式展示
- 如果工作流调用失败，告知用户并提供替代方案
- 所有教学内容必须符合课标要求，不超纲
```

</div>

### 两种方式对比

| 维度 | 插件调用 | 人设指定 |
|------|---------|---------|
| **配置难度** | 简单（3步完成） | 中等（需要编写调用规则） |
| **智能体判断** | 自动识别何时调用 | 手动定义调用条件 |
| **精确度** | 可能误判 | 精确控制 |
| **适合场景** | 功能单一、触发条件明确 | 条件复杂、需要判断 |
| **维护成本** | 低 | 中等（条件变化时需要修改人设） |
| **推荐度** | ⭐⭐⭐⭐⭐（新手首选） | ⭐⭐⭐⭐（进阶推荐） |

> 💡 **建议**：刚开始用"插件调用"（最简单），等需要精确控制时再切换到"人设指定"。

## 多工作流调用

一个智能体可以调用多个工作流，实现复杂的自动化流程。

<div class="case-card">
<span class="case-label">📌 多工作流案例</span>

### 全能备课助手

用户说："帮我准备公开课"

智能体内部流程：
```
用户需求分析
  ↓
判断：用户需要完整公开课材料
  ↓
第1步：调用"教案工作流" → 生成教案
第2步：调用"PPT工作流" → 根据教案生成课件
第3步：调用"评课表工作流" → 生成评课表
  ↓
整合所有结果，按格式展示给用户
```

### 什么时候需要多工作流？

| 场景 | 需要的工作流 | 判断标准 |
|------|-------------|---------|
| 公开课准备 | 教案+课件+评课表 | 用户提到"公开课""赛课" |
| 期末复习 | 知识点梳理+模拟试卷+复习计划 | 用户提到"期末""复习" |
| 活动策划 | 方案策划+海报文案+复盘模板 | 用户提到"活动""策划" |
| 家校沟通 | 话术生成+通知模板+沟通记录 | 用户提到"家长""沟通" |

> 💡 **判断标准**：如果用户的需求涉及2个以上不同类型的输出，就需要多工作流调用。

</div>

## 📋 调用失败排查流程

当工作流没有被正确调用时，按以下流程排查：

<div class="exercise-card">
<span class="exercise-label">🔧 调用失败排查指南</span>

### 第1步：确认工作流已发布

- [ ] 工作流状态显示"已发布"
- [ ] 工作流版本是最新的（修改后需要重新发布）

### 第2步：确认智能体已关联工作流

- [ ] 智能体编辑页面的"插件"或"工作流"模块中能看到该工作流
- [ ] 工作流名称和智能体人设中的调用指令一致

### 第3步：检查调用条件

- [ ] 用户输入包含触发关键词
- [ ] 人设中的调用规则正确（没有拼写错误）
- [ ] 如果使用插件调用，插件名称与触发条件匹配

### 第4步：检查输入参数

- [ ] 工作流的输入参数名称与人设中的参数名一致
- [ ] 参数类型匹配（字符串/数字/选项）
- [ ] 必填参数都已提供

### 第5步：单独测试工作流

- [ ] 在工作流页面点击"测试运行"
- [ ] 输入测试参数，确认工作流能正常执行
- [ ] 如果工作流本身有问题，先修复工作流

</div>

### 常见错误与解决

| 错误 | 原因 | 解决方法 |
|------|------|---------|
| "工作流未找到" | 工作流未发布或名称不匹配 | 确认工作流已发布，名称与人设一致 |
| "参数不匹配" | 输入参数名称或类型错误 | 检查参数名称和类型是否与工作流定义一致 |
| "调用超时" | 工作流执行时间过长 | 优化工作流节点，减少大模型调用次数 |
| "智能体未调用" | 触发条件不匹配 | 在人设中增加更多触发关键词 |
| "结果不完整" | 工作流某节点执行失败 | 单独测试工作流，定位问题节点 |

## 工作流调用的最佳实践

<div class="template-card">
<span class="template-label">📝 最佳实践清单</span>

### 设计阶段

1. **明确触发条件**：每个工作流都应有清晰的触发关键词
2. **参数命名规范**：使用有意义的参数名（如 `subject` 而非 `s`）
3. **设置默认值**：非必填参数设置合理的默认值
4. **错误处理**：在工作流中添加错误处理节点

### 配置阶段

1. **先测试后发布**：每次修改工作流后，先测试再发布
2. **人设简洁明确**：调用规则要简洁，不要过于复杂
3. **预留兜底方案**：如果工作流调用失败，智能体应能直接回答

### 维护阶段

1. **定期测试**：每月至少测试一次所有工作流调用
2. **收集反馈**：记录用户反映的调用问题
3. **持续优化**：根据使用数据调整触发条件和参数

</div>
