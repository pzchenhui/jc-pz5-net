### 变量传递说明

| 变量名 | 来源 | 数据类型 | 去向 |
|--------|------|---------|------|
| `input.topic` | 开始节点用户输入 | 字符串 | 节点1、2、3、4、5 |
| `input.grade` | 开始节点用户输入 | 选项 | 节点2、3、4 |
| `input.subject` | 开始节点用户输入 | 字符串 | 节点2、3、4 |
| `kb_result` | 节点1输出 | 文本段落 | 节点2 |
| `lesson_plan` | 节点2输出 | Markdown文本 | 节点3、4、5 |
| `exercises` | 节点3输出 | Markdown文本 | 节点5 |
| `courseware` | 节点4输出 | Markdown文本 | 节点5 |
| `final_output` | 节点5输出 | Markdown文本 | 结束节点 |

### 提示词编写要点

每个大模型节点的提示词是工作流质量的关键。以下是编写提示词的核心原则：

1. **明确角色**：告诉模型"你是一位有15年教学经验的数学教师"
2. **给出约束**：说明输出格式、字数范围、不包含的内容
3. **提供示例**：给出1-2个输入输出的示例（few-shot提示）
4. **结构化输出**：要求模型按固定格式输出，便于后续节点处理
5. **避免歧义**：不使用"适当""合理"等模糊词汇，用具体数字代替

## 📋 练习一：流程设计（10分钟）

**目标**：用文字或画图设计一个工作流的流程。不需要写代码，只需要理清思路。

### 设计步骤

1. **明确目标**：你想让工作流帮你做什么？（例如：一键出题、批量写评语、生成活动方案）
2. **拆解步骤**：完成这个任务需要哪几个子任务？
3. **设计节点**：每个子任务对应工作流中的一个节点
4. **梳理变量**：节点之间传递什么数据？

### 示例1：张老师的"一键出题"工作流设计

张老师是初中物理教师，每周需要出一次周测题。

**目标**：输入"章节名称 + 难度"，自动出一套测试卷

```
开始（输入：章节"力与运动"，难度"中等"）
  ↓
节点1：知识点提取
  类型：知识库节点
  功能：从教材知识库中检索"力与运动"相关知识点
  输出：知识点列表（牛顿第一定律、力的合成、摩擦力等）
  ↓
节点2：题目生成
  类型：大模型节点
  功能：根据知识点和难度生成题目
  要求：填空题3道 + 选择题4道 + 计算题2道 + 实验题1道
  输出：试卷初稿
  ↓
节点3：答案生成
  类型：大模型节点
  功能：为每道题生成标准答案和评分标准
  输出：参考答案
  ↓
节点4：格式整合
  类型：大模型节点
  功能：将试卷和答案整合为可打印格式
  输出：完整试卷（A4排版，含页眉页脚）
  ↓
结束（输出：可打印的完整试卷）
```

### 示例2：李老师的"批量评语"工作流设计

李老师是五年级班主任，期末需要给50名学生写个性化评语。

**目标**：输入"学生名单（姓名+关键词）"，自动生成50份评语

```
开始（输入：学生名单CSV文件）
  ↓
节点1：数据解析
  类型：代码节点
  功能：读取CSV，提取每位学生的姓名和关键词
  输出：学生信息列表
  ↓
节点2：循环处理（为每个学生生成评语）
  类型：大模型节点 + 循环节点
  功能：根据关键词生成个性化评语
  要求：200-300字，含优点+建议+鼓励
  输出：单个学生评语
  ↓
节点3：格式整合
  类型：大模型节点
  功能：将所有评语整合为表格格式
  输出：完整评语表（姓名+评语）
  ↓
结束（输出：可打印的评语表）
```

### 示例3：王主任的"活动方案"工作流设计

王主任是学校教务主任，需要频繁策划校园活动。

**目标**：输入"活动类型 + 参与人数 + 预算"，自动生成完整活动方案

  类型：大模型节点
  功能：整合方案、安全预案、通知为完整文档

## 📋 练习二：工作流搭建与调试（20分钟）

**目标**：在Coze平台实际操作，将练习一设计的流程搭建出来，并完成调试。

### 详细操作步骤

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">创建工作流</span>

1. 登录 [Coze 平台](https://www.coze.cn)
2. 点击左侧导航栏"工作流"
3. 点击"创建工作流"
4. 填写工作流名称（建议用中文，如"一键备课助手"）
5. 填写工作流描述（简要说明功能）
6. 点击"创建"

<span class="step-number">2</span>
<span class="step-title">设置开始节点</span>

1. 点击"开始"节点
2. 添加输入参数：
   - 点击"添加参数"
   - 参数名：`topic`，类型：字符串，必填：是
   - 参数名：`grade`，类型：选项，选项值：[小学, 初中, 高中]，必填：是
   - 参数名：`subject`，类型：字符串，必填：是
3. 点击"保存"

<span class="step-number">3</span>
<span class="step-title">添加大模型节点</span>

1. 点击"+"按钮，选择"大模型"
2. 选择模型（推荐 GPT-4o 或 Claude 3.5 Sonnet）
3. 编写系统提示词（角色定义）
4. 编写用户提示词（使用 `{{变量名}}` 引用上游变量）
5. 设置输出格式
6. 点击"保存"

<span class="step-number">4</span>
<span class="step-title">连接节点</span>

1. 拖拽连线：从上游节点的输出口连接到下游节点的输入口
2. 确认变量传递正确：点击连线查看传递的变量
3. 测试连接：点击"测试运行"，查看数据流

<span class="step-number">5</span>
<span class="step-title">测试运行</span>

1. 点击右上角"测试"
2. 输入测试参数（如 topic="分数的加减法", grade="小学", subject="数学"）
3. 点击"运行"
4. 查看每个节点的输出预览
5. 如有问题，回到对应节点修改

<span class="step-number">6</span>
<span class="step-title">发布并关联智能体</span>

1. 点击"发布"
2. 前往"智能体"页面
3. 编辑你的智能体
4. 在"能力"或"工具"中添加该工作流
5. 设置触发词（如"帮我备课"）
6. 保存并发布智能体
</div>

### 常见问题与解决

<div class="case-card">
<span class="case-label">❓ 常见问题与解决</span>

#### 问题1：节点之间变量传递失败

**症状**：节点2报"变量未定义"错误，或输出为空。

**原因**：
- 变量名拼写错误（如把 `kb_result` 写成了 `kb_reslut`）
- 前一个节点的输出格式不符合预期
- 变量作用域设置不正确

**解决方法**：
1. 检查变量名拼写，确保前后一致
2. 在节点2的提示词中加入 `{{变量名}}` 语法
3. 使用"测试运行"功能，查看每个节点的输出预览
4. 如果节点1输出是JSON格式，确认节点2正确解析了JSON字段

#### 问题2：大模型节点输出不符合格式要求

**症状**：教案缺少教学目标，或练习题没有答案。

**原因**：
- 提示词不够具体，没有明确输出格式
- 模型"自由发挥"，忽略了部分要求

**解决方法**：
1. 在提示词末尾添加明确的格式要求，如"请严格按照以下格式输出："
2. 给出输出示例（few-shot提示）
3. 将复杂任务拆分为多个节点（如"教案生成"和"练习题生成"分开）
4. 使用结构化输出（JSON格式）替代自由文本

#### 问题3：工作流运行超时

**症状**：点击"运行"后长时间无响应，最终报错"超时"。

**原因**：
- 大模型节点响应慢（模型负载高）
- 节点数量过多（>10个节点）
- 某个节点陷入循环

**解决方法**：
1. 减少节点数量，合并功能相似的节点
2. 选择响应更快的模型（如从GPT-4切换为GPT-3.5）
3. 检查是否有条件分支导致循环
4. 增加超时时间设置（如果有该选项）

#### 问题4：关联智能体后无法调用

**症状**：智能体发布成功，但对话时无法触发工作流。

**原因**：
- 工作流未发布（仅保存未发布）
- 智能体中未添加工作流插件/工具
- 触发条件未满足（如关键词匹配不正确）

**解决方法**：
1. 确认工作流已点击"发布"（不仅是"保存"）
2. 在智能体的"能力"或"工具"配置中添加该工作流
3. 检查触发条件：设置明确的触发词或使用意图识别
4. 在智能体对话中直接输入测试语句验证

#### 问题5：知识库检索结果为空

**症状**：知识库节点返回"未找到相关内容"。

**原因**：
- 知识库中没有上传相关文件
- 上传的文件格式不支持（如扫描件PDF）
- 检索阈值设置过高

**解决方法**：
1. 确认知识库已上传相关文件
2. 将扫描PDF转为可编辑文本后重新上传
3. 降低检索阈值（相似度阈值从0.8调为0.6）
4. 在提示词中添加"如果知识库未返回内容，请基于你的知识回答"
