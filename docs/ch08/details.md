# 8.4 其他细节设置：让行政助手更专业

行政助手的细节设置直接影响公文质量和使用体验。一个专业的行政助手，需要精心配置开场白、公文模板、安全红线等各项细节。

> ⏱️ **预计阅读用时**：约10分钟。

## 开场白设置

好的开场白让行政人员知道智能体"能做什么"，降低使用门槛。

<div class="template-card">
<span class="template-label">📝 推荐开场白模板</span>

```
您好！我是学校行政助手 📋

我可以帮您：
📄 撰写各类公文（通知、报告、总结、请示等）
📝 整理会议纪要
📊 汇总数据报表
📢 撰写新闻稿和宣传文案
📋 制定工作计划

请告诉我您需要什么帮助？
```

### 不同场景的开场白风格

| 场景 | 风格 | 示例 |
|------|------|------|
| **日常办公** | 简洁专业 | "您好！我是行政助手。请告诉我您需要什么类型的公文或服务。" |
| **紧急任务** | 快速响应 | "您好！需要紧急处理什么？请快速告诉我公文类型和关键信息。" |
| **会议管理** | 条理清晰 | "您好！请提供会议基本信息（时间、参会人员、议题），我来帮您整理。" |

</div>

## 细节优化建议

### 1. 公文模板预设

在人设中预置常用公文模板结构：

<div class="template-card">
<span class="template-label">📋 公文模板速查</span>

| 模板类型 | 适用场景 | 关键要素 | 结构 |
|---------|---------|---------|------|
| **通知模板** | 会议通知、活动通知、放假通知 | 时间、地点、对象、事项、要求 | 标题→称呼→正文（背景+事项+要求）→落款→日期 |
| **报告模板** | 工作总结、情况汇报、调研报告 | 背景、过程、结果、建议 | 标题→引言→正文（分点）→结论→落款 |
| **请示模板** | 经费申请、活动申请、人员申请 | 事由、方案、预算、请求 | 标题→主送机关→正文（原因+方案+请求）→落款 |
| **新闻稿模板** | 活动报道、成果宣传 | 5W1H、意义、展望 | 标题→导语→正文→结尾→供稿单位 |
| **纪要模板** | 会议记录、讨论纪要 | 议题、结论、行动项 | 标题→会议信息→议题讨论→决议→行动项 |

</div>

### 2. 知识库配置

| 文件 | 用途 | 推荐数量 | 示例 |
|------|------|---------|------|
| 学校简介和基本信息 | 提供学校基础数据（人数、校训、办学理念） | 1份 | XX学校简介2025版 |
| 公文格式规范文件 | 确保格式正确 | 1-2份 | XX学校公文格式规范 |
| 往年优秀公文范例 | 参考风格和格式 | 3-5份 | 2024年优秀公文汇编 |
| 政策法规摘要 | 常用政策引用 | 1份 | 教育政策法规摘要 |
| 学校管理制度汇编 | 确保内容合规 | 1份 | XX学校管理制度 |

### 3. 插件配置

| 插件 | 用途 | 推荐度 | 配置说明 |
|------|------|--------|---------|
| **联网搜索** | 查找最新政策法规 | ⭐⭐⭐⭐ | 人设中要求"引用政策时标注出处" |
| **文档处理** | 生成Word/PDF文档 | ⭐⭐⭐⭐⭐ | 方便直接导出使用 |
| **代码执行** | 数据统计计算 | ⭐⭐⭐ | 用于成绩统计等场景 |

### 4. 多轮对话配置

| 设置项 | 建议值 | 说明 |
|--------|--------|------|
| **对话轮数** | 10-15轮 | 公文撰写需要收集关键信息并反复修改 |
| **开启对话记忆** | ✅ 开启 | 支持公文逐步修改完善 |
| **话题引导** | 开启 | 偏离时引导回公文处理主题 |

## 安全红线

行政助手涉及公文撰写，必须有明确的安全红线。

<div class="case-card">
<span class="exercise-label">⚠️ 行政安全红线清单</span>

### 5条不可逾越的红线

1. **涉密文件不上传**
   - 人事变动、财务数据、学生个人隐私等敏感内容不交给AI处理
   - 在人设中约束："涉及人事、财务、隐私等内容时，提醒人工处理"

2. **最终审核权在人**
   - 所有公文发布前必须人工审核，AI只生成初稿
   - 每次输出后自动添加："⚠️ 本内容为AI辅助生成初稿，请人工审核后使用"

3. **数据脱敏处理**
   - 学生姓名、身份证号、家庭住址等替换为占位符
   - 在人设中约束："输出中使用[姓名]、[身份证号]等占位符"

4. **保留原始版本**
   - AI生成后保留人工修改记录，便于追溯
   - 建议：使用版本管理，记录每次修改

5. **定期更新知识库**
   - 政策法规变化时及时更新知识库内容
   - 建议：每学期检查一次知识库，更新过期政策

</div>

<span class="exercise-label">📌 安全测试示例</span>

| 测试场景 | 输入指令 | AI正确反应 |

</div>
