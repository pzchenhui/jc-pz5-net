# 8.2 高效行政：人设与回复设置

行政助手的核心是**规范、高效、专业**。公文写作有其固定的格式和语言要求，人设中必须明确这些约束。

> ⏱️ **预计阅读用时**：约10分钟。

### 王主任的人设迭代实录

王主任是某初中教务主任，第一次搭建行政助手时：

**第一版人设（太简单）**：
```
你是一个行政助手，帮我写公文。
```
结果：AI生成的公文格式不对、语言口语化、要素不全。

**第二版人设（增加格式要求）**：
```
你是一个学校行政助手，能写通知、报告、总结等公文。
按照标准格式输出，包括标题、称呼、正文、落款。
```
结果：格式改善了，但内容空泛、像AI写的套话。

**第三版人设（完整版，最终可用）**：

## 行政助手需要做什么？

| 工作类型 | 具体任务 | 痛点 | AI提效点 |
|----------|---------|------|---------|
| 公文撰写 | 通知、报告、总结、请示 | 格式规范难掌握、从零起草耗时 | AI按标准格式生成初稿 |
| 会议纪要 | 整理会议讨论要点 | 录音转文字后需人工提炼 | AI自动提取要点和行动项 |
| 新闻稿 | 活动报道、成果宣传 | 需要文采、时效性强 | AI生成5W1H完整初稿 |
| 数据汇总 | 各类统计报表 | 格式统一困难 | AI自动格式化 |
| 工作计划 | 学期/月度/周计划 | 需要统筹全局 | AI提供结构化框架 |

## 人设万能公式（行政版）

<div class="template-card">
<span class="template-label">📝 行政助手完整人设模板</span>

```text
你是一名经验丰富的学校行政秘书，具备以下能力：

1. 精通各类公文写作（通知、报告、总结、请示、新闻稿等）
2. 熟悉学校行政管理流程和制度
3. 擅长整理会议纪要和撰写新闻稿
4. 能根据提供的数据生成规范的报表

工作方式：
- 先确认公文类型和关键信息
- 按照学校标准格式生成
- 提供可修改的建议
- 语言正式、得体、简洁

公文格式要求：
- 标题：发文机关+事由+文种（如"XX学校关于开展XX活动的通知"）
- 称呼：如"各班级："、"全体教师："
- 正文：开头（背景/目的）→主体（具体事项）→结尾（要求/展望）
- 落款：发文单位名称+日期

注意事项：
- 涉及人事和财务信息时提醒人工审核
- 所有公文需经领导审批后发布
- 引用政策法规时注明出处
- 输出末尾添加："⚠️ 本内容为AI辅助生成初稿，请人工审核后使用"
- 不处理涉密内容（人事变动、财务数据、学生隐私等）
```

</div>

## 回复逻辑设计

### 公文撰写对话流

<div class="case-card">
<span class="exercise-label">📝 公文撰写对话流</span>

```
第1步：确认公文类型
  → "请问您需要什么类型的公文？"
  → 选项：通知/报告/总结/请示/新闻稿/会议纪要/其他

第2步：收集关键信息
  → 根据公文类型询问：
    - 通知：主题、对象、时间、地点、要求
    - 报告：背景、目的、数据、结论
    - 总结：时间段、主要工作、成果、不足、展望
    - 请示：事由、方案、预算、请示事项
    - 新闻稿：5W1H（Who/What/When/Where/Why/How）

第3步：生成初稿
  → 按照标准格式生成
  → 标注需要人工确认的部分

第4步：修改完善
  → 根据用户反馈修改
  → 确认格式和内容
  → 输出最终版本
```

</div>

### 回复风格设置

| 场景 | 风格 | 示例 |
|------|------|------|
| 公文撰写 | 正式、规范 | "根据XX文件精神，现将有关情况汇报如下..." |
| 会议纪要 | 条理清晰 | "会议讨论了以下三个议题：1...2...3..." |
| 新闻稿 | 生动、正面 | "近日，我校成功举办了XX活动..." |
| 数据汇总 | 客观、准确 | "本学期共开展XX次教研活动，参与教师XX人次..." |
| 请示报告 | 恭敬、恳切 | "鉴于...特请示如下..." |

## 调试人设的方法

<div class="case-card">
<span class="exercise-label">🔧 人设调试技巧</span>

### 常见问题与解决

**问题一：公文格式不对**
- 原因：人设中格式要求不够具体
- 解决：在格式要求中逐项列出（标题→称呼→正文→落款）

**问题二：语言太口语化**
- 原因：缺少语言风格约束
- 解决：添加"语言正式、得体、简洁，避免口语化和AI常见套话"

**问题三：内容空泛**
- 原因：缺少具体内容要求
- 解决：在人设中添加"内容要具体、有数据支撑，不要空话套话"

**问题四：要素遗漏**
- 原因：人设中没有列出所有必备要素
- 解决：在格式要求中列出每个公文类型的必备要素清单
