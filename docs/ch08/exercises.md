| **发文字号** | □是 □否 | 如"教务发〔2025〕12号"（可选） | |
| **称呼正确** | □是 □否 | 如"各班级班主任："或"各处室：" | |
| **正文结构** | □是 □否 | 开头（目的/背景）→主体（具体内容）→结尾（执行要求） | |
| **时间地点** | □是 □否 | 事项明确，时间节点清晰 | |
| **执行要求** | □是 □否 | 明确谁在什么时间做什么事 | |
| **落款日期** | □是 □否 | 发文单位和日期完整 | |
| **语言风格** | □是 □否 | 正式、简洁、准确，无口语化表达 | |
| **格式规范** | □是 □否 | 段落分明、编号有序 | |

### 评判标准

| 评分 | 标准 |
|------|------|
| **5分** | 格式规范、内容完整、语言得体，可直接使用或仅需微调 |
| **4分** | 格式基本正确，内容较完整，需要修改部分措辞或补充细节 |
| **3分** | 有基本框架，但格式不够规范或内容缺少关键要素 |
| **2分** | 格式混乱，内容不完整，需要大幅修改 |
| **1分** | 完全不符合公文要求，无法使用 |

### 参考答案与常见问题

<div class="case-card">
<span class="case-label">❓ 参考答案：各类型公文要点</span>

#### 1. 工作总结——合格标准

**应包含的要素**：
- 开篇：时间范围、工作背景
- 成绩部分：量化数据+具体事例（如"本学期组织教研活动12次，参与率95%"）
- 不足部分：客观分析问题（如"部分活动效果未达预期"）
- 展望部分：下学期改进方向

**常见错误**：
- ❌ 只有空话套话，没有具体数据
- ❌ 没有不足分析，全篇"歌功颂德"
- ❌ 展望部分太笼统（"继续努力"不够具体）

#### 2. 工作计划——合格标准

**应包含的要素**：
- 指导思想/工作目标
- 具体任务（分条目列出，含时间节点）
- 保障措施（人员、经费、制度）

**常见错误**：
- ❌ 任务没有时间节点，无法追踪
- ❌ 没有保障措施，计划落地困难
- ❌ 目标不量化，无法评估完成情况

#### 3. 新闻稿——合格标准

**应包含的要素**：
- 标题：简洁有力，含核心信息
- 导语：时间+地点+人物+事件（5W1H）
- 主体：活动过程、亮点、反响
- 结尾：意义或展望

**常见错误**：
- ❌ 标题太长或太短
- ❌ 主体部分流水账，没有重点
- ❌ 语言太正式像公文，或太口语化

#### 4. 会议纪要——合格标准

**应包含的要素**：
- 会议基本信息（时间、地点、主持人、参会人员、记录人）
- 议题及讨论要点
- 决议事项（含责任人和完成时限）

**常见错误**：
- ❌ 只记录"谁说了什么"，没有提炼结论
- ❌ 决议事项没有明确责任人和时间
- ❌ 遗漏重要讨论内容

## 📌 真实案例：行政助手的实际效果

### 案例1：王主任的行政助手使用实录

**背景**：王主任，某市重点中学教务主任，管理5个处室、30+名教师。

**痛点**：
- 每学期要撰写各类公文15-20份（工作计划、总结、通知、报告等）
- 每份公文平均耗时2-3小时，学期初/末高峰期每天工作10+小时
- 公文格式不规范，经常被上级部门退回修改

**搭建行政助手（2025年9月）**：
1. **人设配置**："你是一位有10年学校行政管理经验的行政公文专家，熟悉《党政机关公文处理工作条例》，擅长撰写通知、总结、计划、报告等各类公文。"
2. **知识库**：上传学校历年优秀公文12份（通知4份、总结3份、计划3份、报告2份）
3. **测试迭代**：经过5次调试，确保输出符合学校公文格式

**量化效果（2025年9月-12月数据）**：
- 公文撰写时间：从平均2.5小时→25分钟，**效率提升约83%**
- 公文退回率：从18%→3%，**退回率降低83%**
- 每周行政文书工作时间：从12小时→4小时，**每周节省8小时**
- 同事满意度：行政助手分享给5位同事后，4位表示"显著减少工作量"

> 💡 王主任说："以前写总结要到半夜，现在下午就能搞定，有时间陪家人了。"

<div class="template-card">
<span class="template-label">📝 行政人设模板（3种）</span>

### 模板1：通用行政助手

```
你是一位有10年学校行政管理经验的行政公文专家。

你的核心能力：
1. 熟悉《党政机关公文处理工作条例》和学校公文格式规范
2. 擅长撰写通知、总结、计划、报告、会议纪要、新闻稿等各类公文
3. 了解学校管理流程，能根据公文类型自动匹配合适的格式
4. 语言风格正式、规范、简洁，符合行政公文要求

公文撰写原则：
- 标题规范：使用"关于……的通知/报告/总结"等标准格式
- 结构完整：标题→发文字号→称呼→正文（开头+主体+结尾）→落款→日期
- 用语准确：不使用口语化、网络用语、模糊词汇
- 要素齐全：时间、地点、人物、事项、要求缺一不可

约束条件：
- 不编造数据，需要具体数字时请标注"（待补充）"
- 不确定学校名称时，使用"XX学校"代替
- 如涉及敏感内容，请标注"（请人工审核）"
```

### 模板2：公文审核助手

```
你是一位学校公文审核专家，负责对公文进行格式和内容审核。

审核维度：
1. 格式审核：标题、发文字号、称呼、正文结构、落款日期
2. 内容审核：要素完整性、逻辑性、数据准确性
3. 语言审核：是否正式规范、有无语病错字
4. 合规审核：是否符合学校公文管理要求

输出格式：
- 审核结果：通过/不通过
- 问题清单：逐条列出问题（编号+问题描述+修改建议）
- 总体评价：总体质量评分（1-5分）和改进建议
```

### 模板3：数据报表助手

```
你是一位学校数据分析专家，擅长从各部门数据中提取关键信息并生成分析报告。

核心能力：
1. 能从结构化数据（表格）中提取关键指标
2. 能生成数据分析文字（同比/环比/趋势分析）
3. 能制作数据对比表
4. 能根据数据提出管理建议

输出格式：
- 核心数据概览（关键指标一览表）
- 数据分析（逐项分析，含同比/环比对比）
- 问题与建议（基于数据的管理建议）
- 附件：原始数据来源说明
```

### 挑战1：公文自动审核工作流

搭建一个工作流：输入公文 → AI自动审核格式和内容 → 输出审核报告

```
开始（输入：公文文本）
  ↓
节点1：格式审核
  → 检查标题、称呼、正文结构、落款日期等
  → 输出：格式审核结果（通过/不通过 + 问题列表）
  ↓
节点2：内容审核
  → 检查内容完整性（是否有遗漏要素）
  → 检查语言规范（是否有口语化表达）
  → 输出：内容审核结果
  ↓
节点3：整合报告
  → 整合格式和内容审核结果
  → 输出：完整审核报告 + 修改建议
  ↓
结束（输出：审核报告）
```
