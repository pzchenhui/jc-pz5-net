# 8.2 懂你所需：班主任助手与对话流模式

## 对话流模式是什么？

普通智能体是"一问一答"，对话流模式则是**预先设定好的对话流程**，像填表一样引导用户完成复杂任务。

<div class="case-card">
<span class="case-label">📌 对比案例</span>

### 场景：写期末评语

**普通对话模式**：

```
你：帮我写个评语
AI：请问是哪个学生？什么表现？
你：小明，成绩中上，数学特别好
AI：好的，还有什么特点？
你：很活泼，经常帮助同学
AI：有什么需要改进的吗？
你：有点粗心，作业经常漏题
AI：好的，我来写评语……
```

> 需要来回对话 5-6 轮，才能收集齐信息。

**对话流模式**：

```
第1步：选择学生姓名     [小明  ▼]
第2步：学业表现         [输入框：成绩中上，数学特别好]
第3步：品德表现         [输入框：活泼热情，乐于助人]
第4步：特长爱好         [输入框：数学、篮球]
第5步：需改进的地方     [输入框：粗心，作业漏题]
                        [▶ 生成评语]
```

> 只需填写 5 个字段，一次搞定。
> ⏱️ **节省 3-4 分钟/份评语，批量处理时效率更明显。**

</div>

## 对话流 vs 普通对话：适用场景

| 场景 | 推荐模式 | 原因 |
|------|---------|------|
| 学生评语生成 | **对话流** | 需要收集固定字段信息 |
| 家校沟通记录 | **对话流** | 需要结构化记录 |
| 班会方案设计 | **对话流** | 需要逐步确认要素 |
| 学科问题答疑 | 普通对话 | 自由问答，不可预测 |
| 活动方案策划 | 普通对话 | 需要灵活交互 |

## 对话流的核心优势

| 优势 | 说明 |
|------|------|
| 降低使用门槛 | 用户不需要知道怎么问，按步骤填就行 |
| 信息完整性 | 确保每次都能收集到所有必要信息 |
| 输出一致性 | 同样的输入格式，输出质量稳定 |
| 减少交互轮次 | 一次填写完所有信息，不用来回对话 |
| 适合批量操作 | 连续处理多个学生/多个任务时效率高 |

## 📋 班主任工作流的三种常见模板

<div class="template-card">
<span class="template-label">📝 模板一：学生评语生成流</span>

```
开始
  ↓
【表单】学生基本信息
  - 学生姓名（文本，必填）
  - 班级（下拉选择，必填）
  - 学期（下拉选择，必填）
  ↓
【表单】学业表现
  - 学习成绩概况（文本，必填）
  - 优势学科（文本，选填）
  - 需要提升的学科（文本，选填）
  ↓
【表单】品德与行为
  - 品德表现（文本，必填）
  - 与同学相处情况（文本，选填）
  ↓
【表单】特长与改进
  - 特长爱好（文本，选填）
  - 需要改进的方面（文本，选填）
  ↓
【大模型】生成评语
  - 整合所有输入信息
  - 按评语模板生成
  ↓
结束（输出完整评语）
```
</div>

<div class="template-card">
<span class="template-label">📝 模板二：家校沟通记录流</span>

```
开始
  ↓
【表单】沟通信息
  - 沟通日期（日期选择）
  - 沟通对象（文本：家长姓名）
  - 沟通方式（下拉：电话/微信/面谈/家访）
  ↓
【表单】沟通内容
  - 沟通主题（文本）
  - 沟通要点（文本）
  - 家长反馈（文本）
  ↓
【表单】后续跟进
  - 后续计划（文本）
  - 跟进日期（日期选择）
  ↓
【大模型】生成沟通记录
  ↓
结束
```
</div>
