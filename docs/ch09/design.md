# 9.3 实操：搭建学生评语生成对话流

## 从理解到动手——完整搭建流程

第 9.2 节你已经理解了什么是对话流模式以及为什么班主任助手需要它。现在，**手把手搭建你的第一个对话流工作流**：一个"学生评语生成"工作流。

> ⏱️ **预计阅读用时**：约25分钟 | **本节产出**：一个可运行的学生评语生成工作流，输入学生信息→自动生成评语

<div class="case-card">
<span class="case-label">📌 真实案例</span>

### 李老师的第一次对话流搭建

李老师是小学五年级班主任，从未搭建过工作流。

**搭建前**：
- 期末写 42 份评语，每份约 8 分钟 → 总计 5.6 小时
- 写到后面词穷，评语质量下降

**搭建过程**：
- 跟着本教程的步骤操作 → 约 20 分钟完成搭建
- 测试 3 份评语，全部生成正常

**搭建后**：
- 每份评语只需填写 5 个字段，约 2 分钟
- 42 份 × 2 分钟 = 1.4 小时
- **节省了 4.2 小时，效率提升 75%**

> "比我想象的简单很多。就是填几个字段，连一个大模型节点，就完成了。" —— 李老师

</div>

## 完整搭建步骤

### 步骤一：创建工作流

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">登录并进入工作流页面</span>

1. 登录 [Coze 平台](https://www.coze.cn)
2. 点击左侧导航栏的 **"工作流"**
3. 点击 **"创建工作流"**
4. 选择 **"对话流"** 模式（注意：不是普通工作流）
5. 命名为"学生评语生成"
6. 填写描述："根据学生信息自动生成期末评语"
7. 点击"创建"

<span class="step-number">2</span>
<span class="step-title">添加开始节点和输入参数</span>

点击"开始"节点，添加以下输入参数：

| 参数名 | 显示名称 | 类型 | 必填 | 说明 |
|--------|---------|------|------|------|
| `student_name` | 学生姓名 | 文本 | ✅ | 如"张小明" |
| `grade_info` | 学业表现 | 文本 | ✅ | 如"成绩中上，数学特别好" |
| `moral_info` | 品德表现 | 文本 | ✅ | 如"活泼热情，乐于助人" |
| `strengths` | 特长爱好 | 文本 | ❌ | 如"数学竞赛获奖，篮球" |
| `improvements` | 需改进处 | 文本 | ❌ | 如"粗心，作业漏题" |

> 💡 **提示**：必填字段确保每次都能生成完整评语，选填字段给用户灵活性。

</div>

### 步骤二：添加大模型节点（核心）

这是整个工作流的"大脑"，负责根据输入信息生成评语。

<div class="step-card">
<span class="step-number">3</span>
<span class="step-title">配置大模型节点</span>

1. 点击"+"按钮，选择 **"大模型"**
2. 选择模型（推荐 GPT-4o 或 Claude 3.5 Sonnet）
3. 编写系统提示词（见下方模板）
4. 设置输出变量名：`comment`（评语）

**系统提示词模板**：

```markdown
你是一位有10年经验的优秀班主任，擅长写温暖、真诚、有个性的学生评语。

请根据以下信息，为该同学生成一段100-150字的学期评语：

学生姓名：{{student_name}}
学业表现：{{grade_info}}
品德表现：{{moral_info}}
特长爱好：{{strengths}}
需改进处：{{improvements}}

要求：
1. 用"你"来称呼学生，更有温度
2. 先肯定优点和亮点（2-3条）
3. 委婉地提出改进建议（1条）
4. 以鼓励和期望结尾
5. 语言亲切自然，不要太官方
6. 评语要个性化，避免模板化表述
7. 如果特长爱好为空，跳过特长部分
8. 如果需改进处为空，不提改进建议
```

> 💡 **关键**：第 7、8 条确保选填字段为空时不会出错。

</div>

### 步骤三：连接节点并设置输出

<div class="step-card">
<span class="step-number">4</span>
<span class="step-title">连接节点</span>

1. 从"开始"节点的输出口拖拽连线到"大模型"节点的输入口
2. 确认变量传递正确（点击连线可查看传递的变量）

<span class="step-number">5</span>
<span class="step-title">添加结束节点</span>

1. 点击"+"按钮，选择 **"结束"**
2. 从"大模型"节点的输出口连线到"结束"节点
3. 设置输出变量：选择大模型节点的输出 `comment`
4. 这个输出就是工作流的最终结果

</div>

## 进阶：多评语风格切换

如果你希望生成不同风格的评语，可以在工作流中添加一个"风格选择"字段：

<div class="template-card">
<span class="template-label">📝 技巧：风格选择</span>

### 步骤

1. 在开始节点添加一个新参数：`style`（评语风格）
2. 类型：下拉选择
3. 选项值：
   - `温暖亲切型`（适合低年级）
   - `正式规范型`（适合期末档案）
   - `幽默活泼型`（适合平时鼓励）

### 在大模型节点中根据风格调整

```markdown
评语风格：{{style}}

{% if style == "温暖亲切型" %}
要求：
- 语言亲切可爱，多用 emoji
- 像大姐姐/大哥哥对学生说话
- 多用鼓励话语
{% endif %}

{% if style == "正式规范型" %}
要求：
- 语言正式规范
- 适合写入学生档案
- 避免过于口语化的表达
{% endif %}

{% if style == "幽默活泼型" %}
要求：
- 语言幽默风趣
- 适合日常鼓励
- 可以适当使用轻松的比喻
{% endif %}
```

> 💡 **注意**：Coze 的条件语法可能有所不同，请参考平台的条件分支文档。如果平台不支持条件语法，可以改为创建3个独立的工作流。

</div>

## 进阶：批量评语生成

如果你需要一次性生成多个学生的评语，可以这样优化：

<div class="case-card">
<span class="case-label">📝 技巧：批量处理</span>

### 方法一：使用循环节点

1. 输入一个学生名单（CSV 或表格格式）
2. 使用循环节点，为每个学生执行评语生成
3. 输出所有评语的汇总表

### 方法二：手动多次调用

1. 每次填写一个学生的信息
2. 生成评语后复制保存
3. 继续下一个学生

> 💡 **建议**：学生数量≤50 时，用方法二（手动多次调用）更简单。超过50人时，考虑用循环节点自动化。

</div>

## 常见问题与解决

<div class="case-card">
<span class="exercise-label">❓ 常见问题</span>

### 问题1：评语千篇一律

**症状**：不同学生生成的评语内容相似，缺乏个性。

**解决**：
1. 在系统提示词中强调"评语要个性化，避免模板化表述"
2. 确保输入的学业表现、品德表现等字段内容具体（不要只写"成绩好""品德好"）
3. 在知识库中上传优秀评语范例供参考

### 问题2：评语太长或太短

**症状**：生成的评语超过200字或不到50字。

**解决**：
1. 在提示词中明确字数要求（"100-150字"）
2. 如果仍然不符合，增加更严格的约束（"不超过150字"）
3. 检查输入信息是否过少（信息太少AI无话可说）

### 问题3：选填字段为空时报错

**症状**：不填特长或改进处时，生成的评语出现"undefined"或奇怪的内容。

**解决**：
1. 在提示词中加入条件判断："如果特长爱好为空，跳过特长部分"
2. 为选填字段设置默认值（如"暂无"）
3. 测试时故意不填选填字段，确认不会出错

### 问题4：评语中出现了学生不应该看到的信息

**症状**：评语中提到了"需改进处"的具体内容，但语气太直接。

**解决**：
1. 在提示词中增加约束："改进建议要委婉，不要直接批评"
2. 使用委婉话术模板："如果你能在...方面再努力一些，相信你会更加出色"
3. 生成后人工审核再发布

</div>

工作流测试通过后，需要发布并关联到智能体：

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">发布工作流</span>

1. 点击右上角"发布"按钮
2. 确认发布

<span class="step-number">2</span>
<span class="step-title">关联到智能体</span>

1. 前往"智能体"页面

3. 在"能力"或"工具"中添加刚发布的工作流
4. 设置触发词（如"生成评语"）
5. 保存并发布智能体

<span class="step-number">3</span>
<span class="step-title">测试调用</span>

1. 在智能体对话中输入"生成评语"
2. 确认工作流被正确调用
3. 填写学生信息，检查评语生成结果

</div>
