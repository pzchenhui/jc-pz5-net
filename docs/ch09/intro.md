学校AI工作台是一个**统一的AI服务入口**，将各类智能体整合在一起，让师生按需使用。

```

│  │教学设计 │活动助手 │学科问答  │        │
```

### 各平台在学校场景中的最佳实践

<div class="case-card">
<span class="case-label">💡 平台选择建议</span>

#### 方案一：Coze 团队空间（推荐起点）

**适用学校**：刚开始探索AI、没有技术团队的学校

**搭建步骤**：
1. 在 Coze 创建团队/企业空间
2. 将各类智能体发布到团队空间
3. 按角色分配访问权限
4. 生成工作台页面链接
5. 通过学校工作群分享

**真实案例**：
> 杭州市某小学使用 Coze 团队空间，3周内搭建8个智能体，全校42位教师全部接入，月均使用次数超2000次。

**优点**：
- ✅ 零技术门槛
- ✅ 快速上线（1-2天）
- ✅ 免费额度充足
- ✅ 平台维护，无需技术投入

**缺点**：
- ⚠️ 功能受平台限制
- ⚠️ 数据在第三方平台

**成本**：免费 - 数百元/月（按使用量）

#### 方案二：Dify 自建（技术学校推荐）

**适用学校**：有技术团队、重视数据安全的学校

**搭建步骤**：
1. 服务器部署 Dify 开源平台
2. 创建各类应用和智能体
3. 对接校内系统（教务系统、OA）
4. 统一门户入口
5. 设置权限和审计

**真实案例**：
> 深圳市某中学技术团队部署 Dify，对接校内OA系统，实现单点登录，日均使用次数超500次，数据完全本地化。

**优点**：
- ✅ 数据完全可控
- ✅ 可深度定制
- ✅ 与校内系统对接
- ✅ 开源免费

**缺点**：
- ⚠️ 需要技术人员维护
- ⚠️ 初期投入较大

**成本**：服务器费用 + 人员投入

#### 方案三：智谱清言 + Coze 混合方案

**适用学校**：希望多平台协同、灵活组合的学校

**使用策略**：
- **Coze**：作为主平台，搭建工作流型智能体（备课、活动策划等）
- **智谱清言**：用于需要高质量文本生成的场景（公文撰写、论文辅助）
- **Kimi**：用于长文档分析（整本书解读、政策文件分析）
- **通义千问**：用于需要对接阿里生态的场景（钉钉集成）

**真实案例**：
> 成都市某实验学校采用"主平台+辅助平台"策略：Coze 做智能体搭建，智谱清言做公文润色，Kimi 做课标解读，通义千问做钉钉集成。形成多平台协同的AI工作流。

#### 方案四：Kimi 长文档分析场景

**适用学校**：有大量文档分析需求的学校

**典型场景**：
- 📚 **课标解读**：上传整本课程标准，快速提取关键知识点
- 📋 **政策解读**：上传教育政策文件，提炼要点和行动建议
- 📝 **教材分析**：上传整本教材，生成知识点图谱
- 📊 **试卷分析**：上传历年试卷，分析考点分布

**使用建议**：
> Kimi 的长文本处理能力（支持20万字）是目前最强的，特别适合需要分析大量文档的场景。建议与 Coze 配合使用：Kimi 做文档分析，Coze 做智能体搭建。

</div>

## 权限管理方案

| 角色 | 可访问内容 | 管理权限 |
|------|-----------|---------|
| **校长/管理员** | 全部智能体 + 使用数据 | 用户管理、权限分配、数据查看 |
| **中层干部** | 教学类 + 管理类智能体 | 本部门智能体管理 |
| **学科教师** | 备课、测评、教研类智能体 | 个人智能体管理 |
| **班主任** | 班主任、活动、家校沟通类 | 班级相关智能体 |
| **学生** | 智慧学伴、学科问答 | 仅使用 |
| **家长** | 家校沟通助手 | 仅使用 |

## 📌 平台选择决策树

```
你的学校有技术团队吗？
├── 是 → 选择 Dify 自建（数据安全+深度定制）
└── 否 → 你的主要需求是什么？
    ├── 搭建智能体工作流 → 选择 Coze（功能最全）
    ├── 高质量文本生成 → 选择 智谱清言
    ├── 长文档分析 → 选择 Kimi
    └── 对接钉钉/阿里生态 → 选择 通义千问

推荐组合：
主力平台（Coze 或 Dify）+ 辅助平台（智谱清言/Kimi）
```
