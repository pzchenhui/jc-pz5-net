# 9.2 场景化定制：让AI项目适应学校、学科与班级特点

## 为什么需要场景化定制？

每个学校都不一样：

| 差异维度 | 具体表现 |
|----------|---------|
| 学校规模 | 几百人到几千人 |
| 学科特点 | 文科为主/理科为主/综合 |
| 学生群体 | 城区/农村/寄宿/走读 |
| 教学理念 | 传统/创新/特色课程 |
| 技术基础 | 信息化程度差异大 |

> 🎯 **一刀切的AI方案不如量身定制的方案。**

## 定制的三个层次

### 第一层：学校级定制

<div class="case-card">
<span class="case-label">📌 定制示例</span>

### 农村寄宿制学校 vs 城区走读学校

**农村寄宿制学校的智能体需要**：
- 考虑学生晚自习安排
- 生活管理相关内容
- 家校沟通频率较低的现实
- 留守儿童关怀功能

**城区走读学校的智能体需要**：
- 课后服务相关内容
- 家校高频沟通功能
- 社区资源整合
- 家长参与度高

</div>

### 第二层：学科级定制

<div class="template-card">
<span class="template-label">📝 学科定制要点</span>

| 学科 | 智能体特殊配置 |
|------|---------------|
| **语文** | 阅读理解分析、作文批改、古诗文解析 |
| **数学** | 计算验证、解题步骤拆解、错题分析 |
| **英语** | 语法检查、口语练习、听力材料生成 |
| **科学** | 实验方案设计、科学探究引导 |
| **体育** | 运动方案、体能测试分析、安全教育 |
| **美术** | 作品评价建议、创作灵感引导 |
| **音乐** | 歌曲推荐、乐理知识讲解 |

</div>

### 第三层：班级级定制

| 班级特点 | 智能体调整 |
|----------|-----------|
| 学生基础好 | 增加拓展内容和挑战题 |
| 学生基础弱 | 以基础为主，放慢节奏 |
| 活跃型班级 | 增加互动和活动设计 |
| 安静型班级 | 增加讨论和表达环节 |

在标准人设模板基础上，添加学校特色：

```

【教材版本】和【学校特色】的教学需求。

- 【学校特点1】
- 【学校特点2】
- 【学校特点3】
```

示例：
```

熟悉人教版教材和农村学生的实际情况。

特别注意：
- 学生基础参差不齐，需要分层设计
- 考虑留守儿童的情感关怀
- 教学资源相对有限，方案要务实
```

## 成功案例参考

<div class="case-card">
<span class="case-label">📌 案例一：城区重点小学</span>

**特点**：信息化水平高，家长参与度强

**定制方案**：
- 智慧学伴接入家长端，家长可查看学习建议
- 备课助手整合学校丰富的校本资源
- 活动策划助手侧重科技创新类活动
- 建立校内AI资源库，教师共享优质智能体

</div>

<div class="case-card">
<span class="case-label">📌 案例二：乡镇初级中学</span>

**特点**：教师老龄化，信息化基础弱

**定制方案**：
- 智能体界面极简，操作步骤最少化
- 重点推广备课助手，解决最迫切的需求
- 安排年轻教师作为AI辅导员，一对一帮助老教师
- 先从最简单的功能开始，逐步扩展

</div>

## 持续迭代的机制

| 周期 | 动作 | 负责人 |
|------|------|--------|
| 每周 | 收集使用反馈 | 教研组 |
| 每月 | 分析使用数据，优化智能体 | 教务处 |
| 每学期 | 评估AI推广效果，调整方案 | 校长室 |
| 每学年 | 更新知识库，升级功能 | 技术组 |
