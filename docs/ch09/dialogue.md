# 9.2 懂你所需：班主任助手与对话流模式

## 对话流模式——从"来回问答"到"一次填完"

普通智能体是"一问一答"的对话模式，遇到需要收集多项信息的任务时效率很低。对话流模式则是**预先设定好的结构化流程**，像填表一样引导用户一步步完成，适合班主任工作中的批量任务场景。

> ⏱️ **预计阅读用时**：约20分钟 | **本节产出**：理解对话流模式的适用场景和核心优势

<div class="case-card">
<span class="case-label">📌 对比案例</span>

### 场景：写期末评语

**普通对话模式**（需要 5-6 轮对话）：

```
你：帮我写个评语
AI：请问是哪个学生？什么表现？
你：小明，成绩中上，数学特别好
AI：好的，还有什么特点？
你：很活泼，经常帮助同学
AI：有什么需要改进的吗？
你：有点粗心，作业经常漏题
AI：好的，我来写评语……
```

> 来回问答 5-6 轮才能收集齐信息，写 40 份评语就需要 **200-240 轮对话**。

**对话流模式**（一次填完）：

```
第1步：选择学生姓名     [小明  ▼]
第2步：学业表现         [输入框：成绩中上，数学特别好]
第3步：品德表现         [输入框：活泼热情，乐于助人]
第4步：特长爱好         [输入框：数学、篮球]
第5步：需改进的地方     [输入框：粗心，作业漏题]
                        [▶ 生成评语]
```

> 只需填写 5 个字段，一次搞定。写 40 份评语只需 **40 次填写**，效率提升约 5 倍。
> ⏱️ **每份评语节省 3-4 分钟，批量处理时效率更明显。**

</div>

<div class="case-card">
<span class="case-label">📌 真实案例</span>

### 李老师的期末评语批量处理实录

李老师是小学五年级班主任，需要给 42 名学生写期末评语。

**使用普通对话模式（2024年期末）**：
- 每份评语需要 5-6 轮对话收集信息
- 每份评语平均耗时 8 分钟
- 42 份 × 8 分钟 = **约 5.6 小时**
- 写到后面质量明显下降

**使用对话流模式（2025年期末）**：
- 搭建"学生评语生成"工作流，预设 5 个输入字段
- 每份评语填写约 2 分钟
- 42 份 × 2 分钟 = **约 1.4 小时**
- 评语质量稳定，每份都包含优点、改进建议、鼓励结尾

> ⏱️ **节省了 4.2 小时，效率提升约 75%。而且评语质量更稳定。**

李老师说："以前写到第 20 份就词穷了，现在每份评语质量都差不多。最关键是我只需要填写学生的具体表现，不用反复和 AI 对话收集信息。"

</div>

## 对话流 vs 普通对话：适用场景

判断一个任务是否适合用对话流模式，关键看**是否需要收集固定字段的信息**：

<div class="comparison-card">
<span class="comparison-label">📊 模式选择指南</span>

| 场景 | 推荐模式 | 原因 | 示例 |
|------|---------|------|------|
| 学生评语生成 | **对话流** | 需要收集固定字段（成绩、品德、特长、改进） | 期末评语 |
| 家校沟通记录 | **对话流** | 需要结构化记录（日期、对象、内容、跟进） | 家长沟通存档 |
| 班会方案设计 | **对话流** | 需要逐步确认要素（主题、时间、流程、人员） | 班会策划 |
| 学生档案录入 | **对话流** | 需要固定格式输入 | 学生信息登记 |
| 请假审批记录 | **对话流** | 需要标准化字段 | 请假申请处理 |
| 学科问题答疑 | 普通对话 | 自由问答，问题不可预测 | 数学题解答 |
| 活动方案策划 | 普通对话 | 需要灵活交互和创意碰撞 | 科技节策划 |
| 教育理念探讨 | 普通对话 | 开放式讨论，无固定答案 | 教学方法交流 |

> 💡 **判断标准**：如果需要收集 3 项以上的结构化信息，就适合用对话流模式。

</div>

## 对话流的核心优势

| 优势 | 说明 | 实际效果 |
|------|------|---------|
| **降低使用门槛** | 用户不需要知道怎么问，按步骤填就行 | 新教师也能快速上手 |
| **信息完整性** | 确保每次都能收集到所有必要信息 | 不会遗漏关键字段 |
| **输出一致性** | 同样的输入格式，输出质量稳定 | 40 份评语质量都达标 |
| **减少交互轮次** | 一次填写完所有信息，不用来回对话 | 每份评语节省 3-4 分钟 |
| **适合批量操作** | 连续处理多个学生/多个任务时效率高 | 批量写评语效率提升 5 倍 |
| **数据结构化** | 输入和输出都是结构化数据 | 方便后续统计和分析 |

## 什么是对话流？

对话流（Dialogue Flow）是 Coze 工作流的一种特殊模式，它通过**表单节点**引导用户按步骤输入信息，然后将收集到的信息传递给大模型节点生成结果。

<div class="template-card">
<span class="template-label">📋 对话流的组成要素</span>

| 要素 | 说明 | 示例 |
|------|------|------|
| **开始节点** | 定义工作流的起点 | "学生评语生成"工作流开始 |
| **表单节点** | 引导用户输入信息的步骤 | 输入学生姓名、成绩、品德等 |
| **大模型节点** | 根据输入信息生成内容 | 根据学生信息生成评语 |
| **条件分支**（可选）| 根据不同条件走不同路线 | 根据年级生成不同风格评语 |
| **结束节点** | 输出最终结果 | 输出完整评语 |

### 对话流 vs 普通工作流的区别

| 维度 | 普通工作流 | 对话流 |
|------|-----------|--------|
| **交互方式** | 自动执行，无需用户中途输入 | 需要用户逐步填写表单 |
| **适用场景** | 数据自动处理（如批量生成） | 需要人工输入信息的任务 |
| **用户参与度** | 低（一键执行） | 高（逐步填写） |
| **灵活性** | 高（可配置复杂逻辑） | 中等（表单结构固定） |

</div>

## Coze 中搭建对话流的步骤

<div class="step-card">
<span class="step-number">1</span>
<span class="step-title">进入工作流创建</span>

1. 登录 Coze 平台
2. 点击左侧导航栏"工作流"
3. 点击"创建工作流"
4. 选择"对话流"类型（注意：不是普通工作流）

<span class="step-number">2</span>
<span class="step-title">设计表单节点</span>

每一步添加一个表单字段：
- 字段名：如"学生姓名"
- 字段类型：文本输入/下拉选择/数字/日期
- 是否必填：是/否
- 默认值（可选）：预设常用选项

<span class="step-number">3</span>
<span class="step-title">添加大模型节点</span>

1. 点击"+"按钮，选择"大模型"
2. 编写系统提示词（定义生成规则）
3. 在用户提示词中引用表单字段：`{{字段名}}`
4. 设置输出格式

<span class="step-number">4</span>
<span class="step-title">连接节点并测试</span>

1. 从表单节点的输出口连接到大模型节点的输入口
2. 点击"测试运行"，填写表单字段
3. 检查输出结果是否符合预期
4. 如有问题，回到对应节点修改

<span class="step-number">5</span>
<span class="step-title">发布并关联智能体</span>

1. 点击"发布"
2. 前往"智能体"页面
3. 在"能力"或"工具"中添加该工作流
4. 设置触发词（如"生成评语"）
5. 保存并发布智能体
</div>

## 对话流的设计原则

设计一个好的对话流，需要遵循以下原则：

<div class="exercise-card">
<span class="exercise-label">📋 对话流设计检查表</span>

| 设计原则 | 说明 | 检查项 |
|----------|------|--------|
| **字段精简** | 只收集必要信息，不要过度询问 | 每个字段都用于最终输出？ |
| **顺序合理** | 按逻辑顺序排列字段 | 从简单到复杂？从基本信息到详细信息？ |
| **类型匹配** | 选择合适的字段类型 | 日期用日期选择，性别用下拉？ |
| **必填/选填明确** | 区分必填和选填字段 | 必填字段不遗漏？选填字段有提示？ |
| **有默认值** | 常用选项预设默认值 | 班级、学期等预设当前值？ |
| **输出明确** | 大模型节点的输出格式清晰 | 输出包含所有必要信息？ |

</div>

## 常见问题与解决

<div class="case-card">
<span class="exercise-label">❓ 常见问题</span>

### 问题1：表单字段太多，用户填写麻烦

**症状**：设置了 10+ 个字段，用户填写时间过长。

**解决**：
1. 只保留必填字段，其他设为选填
2. 为常用选项设置默认值
3. 使用下拉选择代替文本输入（减少打字）
4. 将多个相关字段合并为一个（如"学业表现"合并为一个文本框）

### 问题2：生成的内容质量不稳定

**症状**：同样的字段，不同学生生成的评语质量差异大。

**解决**：
1. 优化大模型节点的提示词，增加输出格式约束
2. 在知识库中上传优秀评语范例
3. 增加"评语风格"选项（温暖/正式/幽默）

### 问题3：选填字段为空时生成出错

**症状**：用户不填选填字段，AI生成内容出现"undefined"或空白。

**解决**：
1. 在提示词中加入条件判断："如果{{特长爱好}}为空，则跳过特长部分"
2. 为选填字段设置默认值（如"暂无"）
3. 测试时故意不填选填字段，确认不会出错

</div>
