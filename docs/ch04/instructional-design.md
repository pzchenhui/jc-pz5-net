# 4.3 教学设计助手：巧妙实现课堂创意

## 教案 vs 教学设计

很多人分不清这两个概念：

| 维度 | 教案 | 教学设计 |
|------|------|---------|
| **侧重点** | "教什么"和"怎么教" | "为什么这样教" |
| **内容** | 具体的教学流程 | 教学理念+策略+流程+评价 |
| **深度** | 操作性文档 | 研究性文档 |
| **用途** | 日常上课使用 | 公开课、赛课、教研使用 |

> 📌 **简单说**：教案是"施工图"，教学设计是"建筑设计书"。

<div class="case-card">
<span class="case-label">📌 真实案例</span>

### 李老师的赛课经历

李老师参加市级教学比赛，需要准备一节公开课。

**第一稿（只有教案）**：
- 教研组长看完说："流程有了，但没有设计理念。"

**第二稿（加上教学设计）**：
- 包含了学情分析、设计理念、策略选择依据
- 教研组长说："这才有竞争力。"

**最终结果**：获得市级一等奖。

> 教案告诉你怎么上课，教学设计告诉你**为什么这样上**。

</div>

## 教学设计智能体的核心能力

相比教案生成，教学设计需要：

- 📐 **理论支撑**——建构主义、布鲁姆目标分类等
- 🎯 **策略选择**——为什么选这种教学方式
- 📊 **评价设计**——怎么知道学生学会了
- 🔍 **学情分析**——基于学生实际设计

## 实用技巧

### 技巧一：指定教学流派

<div class="template-card">
<span class="template-label">📝 常用教学流派提示词</span>

| 教学流派 | 提示词示例 | 适用场景 |
|----------|-----------|---------|
| **项目式学习（PBL）** | "请用项目式学习（PBL）的方式设计这节课" | 跨学科、真实问题 |
| **翻转课堂** | "请用翻转课堂的方式设计这节课" | 课前自学+课堂深化 |
| **大概念教学（UbD）** | "请用大概念教学（UbD）的方式设计这节课" | 概念理解、迁移应用 |
| **探究式学习** | "请用探究式学习的方式设计这节课" | 科学实验、发现学习 |
| **合作学习** | "请用合作学习的方式设计这节课" | 小组协作、互动 |
| **混合式教学** | "请设计线上线下混合的教学方案" | 疫情/远程教学 |

</div>

### 技巧二：指定评价方式

```
请设计包含表现性评价的教学方案
请用量规的形式设计评价标准
请设计同伴互评的方案
```

### 技巧三：结合技术工具

```
请在教学设计中融入希沃白板的使用
请设计一个使用平板设备的课堂活动
请融入在线协作工具的使用
```

## 教学设计的质量标准

一份高质量的教学设计应该满足：

<div class="exercise-card">
<span class="exercise-label">✅ 教学设计质量检查清单</span>

- [ ] 学情分析基于真实数据（不是凭空推测）
- [ ] 设计理念有理论支撑（如建构主义、布鲁姆目标分类）
- [ ] 教学目标使用可观测动词（能说出、会操作、能分析）
- [ ] 教学策略选择有明确理由
- [ ] 每个教学环节都有设计意图
- [ ] 评价方案与教学目标对应（目标-评价一致性）
- [ ] 板书设计简洁、逻辑清晰

</div>

## 教案 vs 教学设计：什么时候用哪个？

| 场景 | 推荐类型 | 原因 |
|------|---------|------|
| 日常备课 | 教案 | 快速、实用、够用 |
| 公开课/赛课 | 教学设计 | 需要理论支撑和评价设计 |
| 教研活动 | 教学设计 | 便于同行评议和讨论 |
| 新教师培训 | 两者都做 | 先写教案，再升级为教学设计 |

> 💡 **建议**：日常课用教案（快速实用），公开课用教学设计（有理论深度）。两个智能体可以分别搭建。
