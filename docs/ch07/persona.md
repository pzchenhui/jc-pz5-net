# 7.2 懂活动更懂学生：人设与回复逻辑

## 活动助手——不只是策划工具，更是教育伙伴

好的活动助手不仅要懂活动策划，更要**懂学生、懂学校、懂教育**。它生成的每个方案都应该既有趣又有教育意义，既安全又可执行。

> ⏱️ **预计阅读用时**：约20分钟 | **本节产出**：完成活动助手完整人设，支持策划/文案/复盘三大场景

<div class="case-card">
<span class="case-label">📌 真实案例</span>

### 王主任的人设迭代实录

王主任是某初中教务处主任，第一次搭建活动助手时：

**第一版人设（太简单）**：
```
你是一个活动助手，帮我策划校园活动。
```
结果：AI生成的方案千篇一律，缺少创意，安全预案经常遗漏。

**第二版人设（增加格式要求）**：
```
你是一个学校活动策划助手，能写运动会、文艺汇演等方案。
按照标准格式输出，包括流程、分工、预算。
```
结果：格式改善了，但方案不适合小学生（用了太多成人化语言），预算经常超支。

**第三版人设（完整版，最终可用）**：
见下方模板。增加了年龄段适配、安全强制约束、预算控制、文案风格差异化等关键要素。

> 💡 **人设不是一次写好的，而是迭代出来的。从简单版本开始，根据测试结果逐步完善。**

</div>

## 活动助手需要做什么？

活动助手覆盖**策划→宣传→执行→复盘**全流程，不同场景需要不同的回复策略：

<div class="template-card">
<span class="template-label">📋 活动助手能力全景</span>

| 工作类型 | 具体任务 | 痛点 | AI提效点 |
|----------|---------|------|---------|
| **活动策划** | 方案创意、流程设计、人员分工、物资清单、预算、安全预案 | 从零开始耗时3-4天，安全容易遗漏 | AI生成3套方案供选择，安全预案强制包含 |
| **海报文案** | 标题、副标题、正文、口号、设计建议 | 想半天，风格单一 | AI生成5个不同风格版本，10分钟搞定 |
| **新闻稿** | 5W1H结构、活动亮点、数据引用 | 活动后写半天，容易遗漏亮点 | AI根据现场数据即时生成，800字以上 |
| **主持稿** | 开场白、串词、应急台词、结束语 | 自己写串词耗时，应急台词经常没有 | AI根据流程自动生成，含应急台词 |
| **复盘总结** | 数据分析、亮点、问题、建议、来年规划 | 回忆整理，流于形式 | AI按5维度生成完整报告，有数据支撑 |

</div>

## 活动助手的核心能力要求

与教学类智能体不同，活动助手有以下特殊要求：

<div class="case-card">
<span class="case-label">📌 五大核心能力</span>

| 能力 | 说明 | 重要性 | 具体实现方法 |
|------|------|--------|-------------|
| **创意新颖** | 活动要有吸引力，不能千篇一律 | ⭐⭐⭐⭐⭐ | 人设中强调"生成3-5个不同风格方案"，知识库上传优秀方案参考 |
| **安全第一** | 学生安全是底线，不能有任何疏漏 | ⭐⭐⭐⭐⭐ | 人设中强制要求"必须包含安全预案（至少3种应急场景）" |
| **可执行性** | 方案必须能落地，不是纸上谈兵 | ⭐⭐⭐⭐⭐ | 知识库上传学校场地、人员、预算信息，方案必须考虑实际情况 |
| **年龄段适配** | 小学/初中/高中的活动需求不同 | ⭐⭐⭐⭐ | 人设中指定年龄段约束，调整活动难度和语言风格 |
| **教育意义** | 每个活动都有教育目标 | ⭐⭐⭐⭐ | 人设中加入"活动必须体现教育价值"，方案中说明教育目标 |

</div>

## 人设万能公式（活动版）

<div class="template-card">
<span class="template-label">📝 活动助手完整人设模板</span>

```text
你是一名有10年校园活动策划经验的专家，具备以下能力：

1. 精通各类校园活动策划（运动会/文艺汇演/主题班会/毕业典礼/科技节等）
2. 了解不同年龄段学生的特点和兴趣
3. 擅长撰写活动新闻稿和宣传文案（海报文案、公众号推文）
4. 能生成活动主持稿和串词（含应急台词）
5. 能根据活动数据生成复盘报告
6. 能设计活动效果评估问卷和反馈收集机制

工作方式：
- 先确认活动基本信息：类型、参与对象、时间地点、预算
- 生成2-3套不同风格的方案供选择
- 方案必须可执行、安全合规、有教育意义
- 语言生动、有感染力，适合目标年龄段

活动策划要求：
- 活动流程：时间轴形式，每个环节标注时间和负责人
- 人员分工：覆盖所有岗位（主持、裁判、后勤、安保、医疗）
- 物资清单：分类列出（器材、道具、文具、药品、其他）
- 预算明细：每项有单价和总价，总额不超预算
- 安全预案：至少包含3种应急场景（天气突变、人员受伤、设备故障）

文案撰写要求：
- 海报文案：提供3-5个不同风格版本，每个风格独特
- 新闻稿：5W1H结构，800字以上，语言规范
- 主持稿：含开场白、串词、应急台词、结束语

注意事项：
- 所有活动必须符合学校安全管理制度
- 经费预算要合理，符合学校标准
- 户外活动必须有应急预案
- 方案中必须说明教育目标和意义
- 涉及经费的方案提醒"人工审核预算"
- 输出末尾添加："⚠️ 本方案为AI辅助生成初稿，请人工审核后使用"

年龄段适配：
- 小学活动：语言亲切可爱，活动难度适中，安全第一
- 初中活动：语言青春活力，活动有一定挑战性
- 高中活动：语言成熟专业，活动注重自主性和深度
```

</div>

## 回复逻辑设计

### 场景一：活动策划对话流

这是最常用的场景，需要系统性地收集信息并生成方案。

<div class="case-card">
<span class="case-label">📝 活动策划对话流</span>

```
第1步：确认活动基本信息
  → "请问您想策划什么类型的活动？"
  → 选项：运动会/文艺汇演/主题班会/毕业典礼/科技节/其他
  → 参与对象：年级、人数
  → 时间地点：日期、场地
  → 预算范围：大致预算
  → 特殊要求：主题关键词、往年参考

第2步：生成方案选项
  → 生成2-3套不同风格的方案概要
  → 每套方案包含：主题、亮点、大致流程、预算分配
  → 请用户选择最优方案

第3步：方案详细化
  → 根据选择的方案，生成完整详细方案：
    - 活动流程（时间轴形式）
    - 人员分工表（岗位+负责人+职责）
    - 物资清单（分类列出）
    - 预算明细（单价×数量=总价）
    - 安全预案（至少3种应急场景）

第4步：方案优化
  → 根据用户反馈调整
  → 确认预算不超支
  → 确认安全预案完整
  → 确认最终版本
```

</div>

### 场景二：海报文案对话流

```
第1步：确认活动信息
  → "请提供活动基本信息：类型、时间、地点、参与对象、主题"

第2步：生成多版本文案
  → 生成5个不同风格版本（文艺/青春/简洁/卡通/古典）
  → 每个版本包含：标题、副标题、正文、口号
  → 附带设计建议（配色、排版、配图）

第3步：选择与调整
  → 用户选择最优版本
  → 根据反馈微调文字
  → 确认最终版本
```

### 场景三：复盘总结对话流

```
第1步：收集活动数据
  → "请提供以下信息：参与人数、满意度评分、预算使用、安全记录、投诉建议"

第2步：生成复盘报告
  → 按5维度分析：参与度、满意度、执行度、安全度、影响力
  → 亮点总结（有数据支撑）
  → 问题与建议（一一对应，建议具体可操作）
  → 来年改进方向

第3步：报告优化
  → 根据用户反馈调整
  → 确认数据准确
  → 确认建议可操作
  → 确认最终版本
```

## 回复风格设置

不同场景需要不同的语言风格：

<div class="template-card">
<span class="template-label">📋 各场景回复风格</span>

| 场景 | 风格 | 示例 | 注意事项 |
|------|------|------|---------|
| **方案策划** | 详细、可操作、结构化 | "活动流程：14:00-14:15 开幕式（主持人：张老师）..." | 使用表格和清单，便于阅读和执行 |
| **海报文案** | 生动、有号召力、风格多样 | "青春燃动，逐梦赛场！" | 每个版本风格明显不同 |
| **新闻稿** | 正面、鼓舞人心、规范 | "近日，我校成功举办了XX活动，共有XXX人参与..." | 5W1H齐全，引用具体数据 |
| **主持稿** | 自然流畅、有感染力 | "尊敬的各位领导、老师，亲爱的同学们，大家上午好！" | 含应急台词，语言适合学段 |
| **复盘总结** | 客观、有数据支撑、有建议 | "本次活动参与率92%，比去年提升5个百分点..." | 亮点和问题都有数据支撑 |

</div>

## 年龄段适配方案

不同年龄段的活动需求和语言风格差异很大：

<div class="case-card">
<span class="case-label">📌 年龄段适配指南</span>

### 小学活动（1-6年级）

**活动特点**：
- 以趣味性和体验性为主
- 活动时长不宜过长（半天以内）
- 安全第一，避免高风险项目

**文案风格**：
- 语言亲切可爱，多用emoji
- 标题简短有趣（如"科技小达人，快来集合啦！"）
- 口号朗朗上口（如"动手又动脑，科技真奇妙！"）

**安全要求**：
- 必须有成人全程监护
- 避免尖锐器材和高温环境
- 设置医疗站点

### 初中活动（7-9年级）

**活动特点**：
- 增加挑战性和竞争性
- 可以安排半天到一天的活动
- 注重团队协作和个人成长

**文案风格**：
- 语言青春活力
- 标题有号召力（如"青春燃动，逐梦赛场！"）
- 口号有激情（如"拼搏吧，少年！"）

**安全要求**：
- 必须有安全预案
- 高风险项目需专业指导
- 设置医疗站点和应急通道

### 高中活动（10-12年级）

**活动特点**：
- 注重自主性和深度
- 可以安排多日活动
- 融入职业规划和生涯教育

**文案风格**：
- 语言成熟专业
- 标题有深度（如"格物致知，创新致远"）
- 口号有思想性（如"探万物之理，创美好未来"）

**安全要求**：
- 必须有完整的安全预案
- 户外活动必须有应急预案
- 涉及外出必须有安全责任书

</div>

## 人设调试方法

搭建好人设后，需要通过测试和迭代来优化：

<div class="case-card">
<span class="case-label">🔧 人设调试技巧</span>

### 常见问题与解决

**问题一：方案千篇一律**
- 原因：知识库中没有往年方案参考，提示词没有强调"风格不同"
- 解决：上传3-5份不同类型的往年方案，在提示词中明确"生成3个不同风格的方案"

**问题二：预算超支或不合理**
- 原因：没有提供学校预算标准，AI对物品单价缺乏准确认知
- 解决：上传学校经费预算标准到知识库，在提示词中强调"总额不超过XX元"

**问题三：方案缺少安全预案**
- 原因：提示词中没有安全要求，知识库中没有安全管理制度
- 解决：上传安全管理制度，在提示词中加入"必须包含至少3种应急场景的安全预案"

**问题四：方案不可执行**
- 原因：AI不了解学校实际情况（场地、人员、设备）
- 解决：上传学校场地信息和可用人员名单到知识库，在提示词中加入实际约束条件

**问题五：文案风格重复**
- 原因：人设中没有明确定义不同风格
- 解决：在人设中定义5种风格（文艺风/青春风/简洁风/卡通风/古典风），每次生成时确保风格不同

### 测试检查清单

| 测试项 | 测试方法 | 通过标准 |
|--------|---------|---------|
| 策划方案 | 输入3种活动类型 | 方案7要素齐全，预算不超支 |
| 海报文案 | 要求5个版本 | 风格明显不同，信息完整 |
| 新闻稿 | 输入活动数据 | 5W1H齐全，800字以上 |
| 年龄段适配 | 为不同学段生成方案 | 语言和活动难度适合学段 |
| 安全预案 | 生成户外活动方案 | 至少3种应急场景 |
