# 7.1 了解校园活动助手智能体

## 校园活动——教育的另一种课堂

校园活动是课堂教学的延伸，**让学生在实践中学习、在体验中成长**。但活动策划也是教师最耗时的工作之一。

> ⏱️ **预计阅读用时**：约20分钟。

<div class="case-card">
<span class="case-label">📌 真实痛点</span>

### 活动策划的真实痛点

**痛点一：从零开始太耗时**

刘老师是某初中科技节负责人，往年自己策划：
1. 查往年资料（2小时）→想主题（1小时）→设计方案（3小时）→列流程（2小时）→排人员分工（1小时）
2. 反复修改3-4稿，征求各部门意见
3. **总耗时：约3-4天**

**痛点二：安全问题容易遗漏**

2024年某校运动会，因未准备雨天备选方案，活动被迫取消，损失了前期所有准备工作。

**痛点三：宣传文案质量不稳定**

- 有的老师文采好，海报文案有吸引力
- 有的老师不擅长文案，海报千篇一律
- 缺乏统一的宣传标准

**痛点四：活动后复盘流于形式**

- 多数活动"办完就完了"，没有系统复盘
- 经验没有沉淀，下次还是从头开始
- 缺乏数据支撑的改进建议

<span class="case-label">📌 真实案例</span>

### 李老师的校园文艺汇演AI辅助实践

**背景**：李老师是小学音乐教师，负责年度"六一"文艺汇演（600人参与）。

**传统方式（2024年）**：
- 策划方案：自己从头写，反复修改 → 约3天
- 海报文案：自己想+找人帮忙 → 约2小时
- 主持稿：自己写串词 → 约2小时
- 新闻稿：活动后写半天 → 约2小时
- 复盘总结：回忆整理 → 约1小时
- **总计约4天**

**使用AI活动助手后（2025年）**：
- 策划方案：AI生成3套方案 → 选择最优 → 约1小时
- 海报文案：AI生成5个版本 → 选择最优 → 约10分钟
- 主持稿：AI根据节目单生成 → 约15分钟
- 新闻稿：AI根据现场数据生成 → 约15分钟
- 复盘总结：AI分析问卷数据 → 约20分钟
- **总计约2小时，效率提升约97%**

> "以前策划文艺汇演要熬好几个周末，现在AI帮我从'从零开始'变成'审核修改'，效率完全不同。而且AI生成的安全预案比我想的还周到。" —— 李老师

## 活动助手能帮你做什么？

## 活动助手的特殊之处

与教学类智能体不同，活动助手需要特别关注以下方面：

<div class="case-card">
<span class="case-label">📌 活动助手核心要求</span>

### 五大核心能力

| 特点 | 说明 | 重要性 | 具体实现方法 |
|------|------|--------|-------------|
| **创意新颖** | 活动要有吸引力，不能千篇一律 | ⭐⭐⭐⭐⭐ | 人设中强调"生成3-5个不同风格方案" |
| **安全第一** | 学生安全是底线，不能有任何疏漏 | ⭐⭐⭐⭐⭐ | 人设中强制要求"必须包含安全预案" |
| **可执行性** | 方案必须能落地，不是纸上谈兵 | ⭐⭐⭐⭐⭐ | 知识库上传学校场地、人员、预算信息 |
| **有感染力** | 文案要有号召力，能激发参与欲 | ⭐⭐⭐⭐ | 人设中预设"口号朗朗上口"要求 |
| **教育意义** | 每个活动都有教育目标 | ⭐⭐⭐⭐ | 人设中加入"活动必须体现教育价值" |

### 活动助手 vs 教学助手的区别

| 维度 | 教学助手（备课/学伴） | 活动助手 | 差异原因 |
|------|---------------------|---------|---------|
| **核心能力** | 知识准确、课标合规 | 创意新颖、安全合规 | 教学活动更重知识，活动更重创意 |
| **输出类型** | 教案、试题、课件 | 方案、文案、主持稿、报告 | 活动输出更多样化 |
| **时效性** | 长期可用 | 每次活动不同，需要定制 | 活动是一次性的，教案可复用 |
| **安全要求** | 内容安全 | 人身安全+内容安全 | 活动涉及学生户外活动 |
| **创意要求** | 稳定、规范 | 新颖、有吸引力 | 活动需要吸引学生参与 |

</div>
