# 7.3 活动创意策划智能体——从灵感到可执行方案

## 活动策划——从创意到可执行方案

活动创意策划是活动助手最核心的功能。一个好的活动方案需要**创意新颖、流程合理、安全合规、可落地执行**。

> ⏱️ **预计阅读用时**：约20分钟 | **本节产出**：搭建活动策划工作流，能一键生成完整活动方案

<div class="case-card">
<span class="case-label">📌 真实案例</span>

### 刘老师的科技节策划实录

刘老师是某初中科技节负责人，管理1200人规模的校园活动。

**传统方式（2024年）**：
1. 查往年资料→想主题→设计方案→列流程→排人员→算预算
2. 反复修改3-4稿，征求各部门意见
3. 耗时：约**3-4天**

**使用AI活动智能体后（2025年）**：
1. 输入活动信息（类型、人数、预算、学科参与）
2. AI生成3套不同风格的方案供选择
3. 选择最优方案，微调人员分工和预算细节
4. 耗时：约**1小时**

> ⏱️ **节省了约3天，效率提升约10倍。而且AI生成的方案比往年更系统、更安全。**

刘老师说："AI不是替代我策划，而是把'从零开始'变成了'审核修改'。以前我从空白文档开始写，现在从3套方案中选最优，起点完全不同。"

</div>

### 李老师的"六一"文艺汇演策划

李老师是小学音乐教师，负责年度"六一"文艺汇演。

**AI辅助策划过程**：
1. **创意生成**：输入"六一文艺汇演、600人、预算8000元、童话主题" → AI生成3个主题方案
2. **方案选择**：李老师选择"童话城堡"主题
3. **方案详细化**：AI自动生成完整方案（节目单、彩排表、舞台布置、人员分工、预算、安全预案）
4. **人工调整**：修改节目顺序、调整预算分配

**效果**：
- 策划时间从3天→1小时
- 方案包含往年遗漏的安全预案和应急预案
- 预算分配更合理（AI根据往年数据给出建议比例）

## 策划工作流搭建——从输入到输出的自动化

活动策划工作流是整个活动助手的核心。它需要完成从"模糊需求"到"完整方案"的转换。

### 工作流设计总览

<div class="template-card">
<span class="template-label">📋 活动策划工作流全景</span>

```
┌─────────────────────────────────────────────────┐
│ 开始节点                                          │
│ 输入参数：                                        │
│   - activity_type（活动类型）：运动会/文艺汇演/等  │
│   - participants（参与人数）：数字                │
│   - budget（预算）：数字                          │
│   - theme（主题关键词）：字符串（可选）            │
│   - venue（场地）：字符串                         │
└─────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────┐
│ 节点1：知识库检索——往年方案参考                    │
│ 从知识库中检索类似活动的方案                      │
│ 输出：往年方案摘要（2-3份）                       │
└─────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────┐
│ 节点2：创意生成                                   │
│ 生成3个不同风格的活动主题和方案概要               │
│ 每个方案包含：主题、亮点、大致流程、预算分配       │
│ 输出：3套方案概要                                 │
└─────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────┐
│ 节点3：方案详细化（用户选择后）                    │
│ 根据选择的方案，生成完整详细方案：                │
│   - 活动流程（时间轴形式）                        │
│   - 人员分工表（岗位+负责人+职责）                │
│   - 物资清单（分类列出）                          │
│   - 预算明细（单价×数量=总价）                    │
│   - 安全预案（至少3种应急场景）                   │
│ 输出：完整活动方案文档                            │
└─────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────┐
│ 节点4：格式整理输出                               │
│ 整合所有内容为统一的Markdown文档                  │
│ 添加封面、目录、页眉页脚                          │
│ 输出：可打印的活动方案文档                        │
└─────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────┐
│ 结束节点                                          │
│ 输出：完整活动方案文档                            │
└─────────────────────────────────────────────────┘
```

</div>

### 变量传递说明

| 变量名 | 来源 | 数据类型 | 去向 |
|--------|------|---------|------|
| `input.activity_type` | 开始节点 | 字符串 | 节点1、2、3 |
| `input.participants` | 开始节点 | 数字 | 节点2、3 |
| `input.budget` | 开始节点 | 数字 | 节点2、3 |
| `input.theme` | 开始节点 | 字符串 | 节点2 |
| `kb_schemes` | 节点1 | 文本 | 节点2 |
| `scheme_options` | 节点2 | 文本 | 节点3 |
| `detailed_scheme` | 节点3 | 文本 | 节点4 |
| `final_document` | 节点4 | Markdown | 结束节点 |

### 节点2：创意生成的提示词设计

创意生成是工作流质量的关键。以下是推荐的大模型节点提示词：

<div class="template-card">
<span class="template-label">📝 创意生成提示词模板</span>

```text
你是一位有10年校园活动策划经验的专家。

请根据以下信息生成3个不同风格的活动方案概要：
- 活动类型：{{input.activity_type}}
- 参与人数：{{input.participants}}人
- 预算：{{input.budget}}元
- 主题关键词：{{input.theme}}

参考往年方案：
{{kb_schemes}}

要求：
1. 每个方案风格不同（如：传统型、创新型、互动型）
2. 每个方案包含以下要素：
   - 活动名称（有吸引力，15字以内）
   - 活动主题（一句话概括，20字以内）
   - 核心亮点（3个最具吸引力的特色）
   - 大致流程（4-6个主要环节）
   - 预算分配比例（各项占比百分比）
3. 方案必须适合目标年龄段学生
4. 预算分配总额不超过{{input.budget}}元
5. 必须包含安全考虑

输出格式：
---
### 方案一：[方案名称]
**主题**：[一句话主题]
**亮点**：[3个亮点]
**流程**：[4-6个环节]
**预算分配**：[各项占比]
---
（方案二、三同理）
```

</div>

### 节点3：方案详细化的提示词设计

```text
你是一位有10年校园活动策划经验的专家。

请根据以下选择的方案概要，生成完整的活动方案文档：

活动信息：
- 活动类型：{{input.activity_type}}
- 参与人数：{{input.participants}}人
- 预算：{{input.budget}}元
- 场地：{{input.venue}}
- 主题关键词：{{input.theme}}

选定方案概要：
{{scheme_options}}

请生成包含以下完整要素的活动方案：

## 一、活动基本信息
- 活动名称、时间、地点、参与对象

## 二、活动流程（时间轴形式）
- 每个环节标注：时间、内容、负责人、注意事项

## 三、人员分工表
- 以表格形式列出所有岗位：岗位名称、人数、负责人、具体职责

## 四、物资清单
- 分类列出：器材类、道具类、文具类、药品类、其他
- 每项标注：数量、单价、总价

## 五、预算明细
- 以表格形式列出所有费用项目
- 总额不超过{{input.budget}}元

## 六、安全预案
- 场景1：天气突变的应急措施
- 场景2：人员受伤的应急处理
- 场景3：设备故障的替代方案
- 紧急联系方式列表

输出要求：
- 使用Markdown格式
- 内容具体可操作，不要空话套话
- 预算明细使用表格
- 安全预案必须包含至少3种应急场景
```

## 知识库配置

活动策划的质量很大程度上取决于知识库中往年方案的参考质量。

<div class="case-card">
<span class="case-label">📋 知识库文件清单</span>

| 文件类型 | 用途 | 推荐数量 | 文件格式 |
|----------|------|---------|---------|
| **往年活动方案** | 参考风格、格式、预算分配 | 3-5份 | PDF/Word |
| **学校场地信息** | 了解可用场地、容量、设备 | 1份 | PDF/Word |
| **安全管理制度** | 确保方案符合安全要求 | 1份 | PDF |
| **经费预算标准** | 控制预算在合理范围 | 1份 | PDF/Excel |
| **学校简介** | 让AI了解学校文化和特色 | 1份 | PDF/Word |

**上传建议**：
- 往年方案选择不同类型的（运动会、文艺汇演、主题班会各1-2份）
- 确保文件文字可复制（扫描件需OCR转换）
- 单个文件不超过10MB

## 不同活动类型的策划要点

不同类型活动的策划重点不同，AI需要针对性调整：

<div class="template-card">
<span class="template-label">📋 各类型活动策划要点</span>

### 运动会

**核心要素**：
- 项目设置（田赛+径赛+趣味项目）
- 场地规划（各比赛区域分布）
- 裁判和工作人员安排
- 开幕式和闭幕式流程
- 奖品和证书准备

**安全重点**：
- 运动伤害应急预案
- 高温/雨天备选方案
- 医疗站点设置

### 文艺汇演

**核心要素**：
- 节目征集和选拔
- 彩排时间安排
- 舞台和音响灯光布置
- 主持人和串词
- 观众席安排

**安全重点**：
- 舞台安全检查
- 人员密集区域疏导
- 用电安全

### 主题班会

**核心要素**：
- 主题确定和活动设计
- 学生参与环节
- 互动游戏设计
- 总结发言

**安全重点**：
- 室内活动相对安全
- 注意学生心理安全（避免公开批评等）

### 科技节/艺术节

**核心要素**：
- 展览展示区规划
- 比赛项目设置
- 嘉宾邀请
- 作品征集和评选

**安全重点**：
- 实验类活动安全
- 电器设备安全
- 人流密集区域管理

### 毕业典礼

**核心要素**：
- 典礼流程设计
- 颁奖环节
- 家长邀请
- 纪念册/纪念品准备

**安全重点**：
- 大型活动安保
- 应急预案

## 常见问题与解决

<div class="case-card">
<span class="case-label">❓ 常见问题</span>

### 问题1：AI生成的方案千篇一律

**症状**：每次生成的方案内容相似，没有创意感。

**原因**：
- 知识库中没有往年方案参考
- 提示词没有强调"风格不同"

**解决**：
1. 上传3-5份不同类型的往年方案到知识库
2. 在提示词中明确"生成3个不同风格的方案"
3. 加入主题关键词约束（如"科技"、"环保"等）

### 问题2：方案预算超支或不合理

**症状**：AI生成的预算总额超过给定预算，或某项费用明显不合理。

**原因**：
- 没有提供学校预算标准
- AI对物品单价缺乏准确认知

**解决**：
1. 上传学校经费预算标准到知识库
2. 在提示词中强调"总额不超过XX元"
3. 人工审核预算明细，调整不合理项

### 问题3：方案缺少安全预案

**症状**：AI生成的方案没有安全相关内容。

**原因**：
- 提示词中没有安全要求
- 知识库中没有安全管理制度

**解决**：
1. 上传学校安全管理制度到知识库
2. 在提示词中加入"必须包含至少3种应急场景的安全预案"
3. 在人设中添加安全约束

### 问题4：方案不可执行

**症状**：方案看起来很完美，但实际操作不了（如时间冲突、人手不够）。

**原因**：
- AI不了解学校实际情况（场地、人员、设备）
- 方案过于理想化

**解决**：
1. 上传学校场地信息和可用人员名单到知识库
2. 在提示词中加入实际约束条件
3. 人工审核时重点关注可行性

</div>
