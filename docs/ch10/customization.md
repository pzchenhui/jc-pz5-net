# 10.2 场景化定制：让AI项目适应学校、学科与班级特点

## 为什么需要场景化定制？

每个学校都不一样：

| 差异维度 | 具体表现 |
|----------|---------|
| 学校规模 | 几百人到几千人 |
| 学科特点 | 文科为主/理科为主/综合 |
| 学生群体 | 城区/农村/寄宿/走读 |
| 教学理念 | 传统/创新/特色课程 |
| 技术基础 | 信息化程度差异大 |

> 🎯 **一刀切的AI方案不如量身定制的方案。**

## 定制的三个层次

### 第一层：学校级定制

### 三种典型学校的AI定制方案

**1. 农村寄宿制学校**

| 特点 | 定制方案 |
|------|---------|
| 留守儿童多 | 智能体增加情感关怀话术 |
| 晚自习集中 | 智慧学伴适配晚自习场景 |
| 家校沟通频率低 | 班主任助手生成月度沟通报告 |
| 教学资源有限 | 备课助手侧重低成本教学方案 |
| 网络条件有限 | 选择离线可用的平台方案 |

**2. 城区走读学校**

| 特点 | 定制方案 |
|------|---------|
| 家长参与度高 | 智慧学伴开放家长端 |
| 课后服务丰富 | 活动助手侧重课后活动设计 |
| 社区资源丰富 | 智能体整合社区资源库 |
| 信息化水平高 | 可对接校内系统，建设统一工作台 |

**3. 民办学校**

| 特点 | 定制方案 |
|------|---------|
| 特色课程多 | 智能体融入校本课程 |
| 品牌意识强 | 统一AI智能体视觉风格 |
| 家长期望高 | 增加学习反馈功能 |
| 教师流动大 | 智能体配置标准化，便于交接 |

### 第二层：学科级定制

<div class="template-card">
<span class="template-label">📝 学科定制要点</span>

| **语文** | 阅读理解分析、作文批改、古诗文解析 | Coze + 智谱清言 | 作文批改：上传→AI评分→批注→修改建议 |

| **科学** | 实验方案设计、科学探究引导 | Coze + 通义千问 | 实验方案：课题→材料→步骤→安全提示 |

| **音乐** | 歌曲推荐、乐理知识讲解 | Coze | 乐理讲解：概念→示例→练习→检测 |

### 第三层：班级级定制

| 班级特点 | 智能体调整 |
|----------|-----------|
| 学生基础好 | 增加拓展内容和挑战题 |
| 学生基础弱 | 以基础为主，放慢节奏 |
| 活跃型班级 | 增加互动和活动设计 |
| 安静型班级 | 增加讨论和表达环节 |
| 留守儿童多 | 增加关怀和鼓励话语 |
| 国际班 | 双语输出，跨文化内容 |

## 成功案例参考

<div class="case-card">
<span class="case-label">📌 案例一：城区重点小学</span>

**学校特点**：信息化水平高，家长参与度强，学生1200人

**使用平台**：Coze 团队空间 + 智谱清言辅助

**定制方案**：
- 智慧学伴接入家长端，家长可查看学习建议
- 备课助手整合学校丰富的校本资源（校本课程PDF上传知识库）
- 活动策划助手侧重科技创新类活动
- 行政助手使用智谱清言生成高质量公文，再导入Coze智能体
- 建立校内AI资源库，教师共享优质智能体

**成效**：
- 3个月内教师使用率达 88%
- 家长满意度 95%
- 被评为"市级教育信息化示范校"

</div>

<div class="case-card">
<span class="case-label">📌 案例二：乡镇初级中学</span>

**成效**：
- 6个月内教师使用率达 65%
- 老教师从抗拒到接受，备课效率提升 50%
- 年轻教师成为AI骨干力量

<div class="case-card">

## 多平台协同工作流案例

<div class="template-card">
<span class="template-label">🔗 多平台协同最佳实践</span>

### 场景：备课全流程

**Step 1：Kimi 课标分析**
> 上传整本课程标准 → Kimi 提取知识点和能力要求 → 输出课标解读报告

**Step 2：Coze 教案生成**
> 导入课标解读报告到 Coze 知识库 → 搭建备课工作流 → 自动生成教案

**Step 3：智谱清言 教案润色**
> 将 Coze 生成的教案复制到智谱清言 → 润色语言 → 优化教学设计

**Step 4：Coze 课件大纲**
> 将润色后的教案导入 Coze → 调用课件生成工作流 → 输出课件大纲

**Step 5：通义千问 课件制作**
> 使用通义千问的PPT生成功能 → 根据大纲制作课件 → 下载PPT文件

**总耗时**：约 30 分钟（传统方式约 3 小时）

</div>
