# 📝 第5章练习：学科助手

## 练习目标

搭建面向学生的智慧学伴智能体，能引导学生思考而非直接给答案。

## 练习前准备

在开始实操之前，请确保你已完成以下准备：

- [ ] 完成第 5.2-5.5 节的学习
- [ ] 已注册并登录至少一个 AI 平台
- [ ] 准备好对应学科的课程标准和教材电子版
- [ ] 准备 5-10 个学生常见问题（或回忆你自己小时候会问的问题）

## 📌 参考答案与评判标准

### 智慧学伴评判标准

<div class="case-card">
<span class="case-label">📋 参考答案</span>

**优秀的智慧学伴应具备以下特征**：

1. **引导式回复**：不直接给答案，通过提问和提示引导学生思考
   - 检查方法：直接要答案时，AI 应拒绝并引导
   
2. **知识准确**：学科知识回答准确率≥95%
   - 检查方法：审核 10 个知识点问题，错误≤1 个
   
3. **安全合规**：安全红线测试 5 项全部通过
   - 检查方法：按上文"安全过滤测试表"逐项测试
   
4. **学段适配**：语言、深度、引导方式都符合目标学段
   - 检查方法：同一知识点测试小学/初中/高中三个版本
   
5. **有温度**：有鼓励机制，不同情境有不同回复
   - 检查方法：模拟 5 种情境（答对/答错/困惑/进步/放弃）
   
6. **开场友好**：开场白简洁、温暖、有行动引导
   - 检查方法：不超过 5 行，包含功能说明和提问引导

**合格线**：上述 6 项全部达标。

</div>

### 王老师搭建英语学伴的经历

### 李老师搭建数学学伴的"翻车"与修复

李老师，某初中数学教师，第一次搭建数学学伴。

**第一次搭建的问题**：
- 人设中没有明确"不直接给答案"→ 学生一问题目，学伴就把答案给了
- 知识库只上传了教材目录 → 检索不到具体内容
- 没有设置安全过滤 → 学生问了与学习无关的问题，学伴也回答了

**修复过程**：
1. 在人设中添加："绝对不直接给答案，必须引导学生自己思考"
2. 重新整理知识库：上传完整的教材内容 + 课标 + 典型例题
3. 开启平台安全过滤，并在人设中添加安全约束

**修复后的效果**：
- 引导效果从 2/5 提升到 5/5
- 知识准确性从 70% 提升到 95%
- 安全过滤从 2/5 提升到 5/5

李老师说："第一次搭建翻车很正常，关键是发现问题后要针对性修复。建议第一次搭建完一定要按检查清单逐项测试。"

### 赵老师的语文学伴搭建之旅

赵老师，某小学语文教师，教四年级。

**搭建动机**：班上 45 名学生，阅读理解能力差异大。有些学生课后家长不会辅导，有些学生遇到阅读题就放弃。

**搭建过程**：
1. **人设编写**（20 分钟）：
   - 角色：语文阅读学习伙伴
   - 引导方式：不直接告诉文章中心思想，而是引导学生找出关键句
   - 鼓励机制：用"你真会思考！"等鼓励话术

2. **知识库配置**（40 分钟）：
   - 上传四年级语文教材（上册 + 下册）
   - 上传阅读理解方法指导（10 篇）
   - 上传典型阅读理解错题集（20 道）

3. **安全设置**（5 分钟）：
   - 不代写作文
   - 不直接给阅读理解答案
   - 限制与语文无关的话题

**使用效果（两个月后）**：
- 阅读理解平均正确率：从 58% → 76%（+18%）
- 课后作业完成率：从 82% → 94%（+12%）
- 学生主动阅读意愿：从 35% → 68%（+33%）

赵老师说："最让我感动的是，班上最不爱说话的一个孩子，竟然主动跟 AI 学伴讨论课文。他说'它不会嫌我笨'。"
